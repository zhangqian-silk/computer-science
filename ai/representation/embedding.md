# Embedding：离散对象的连续向量表示

> 相关文献：
> - Bengio et al. (2003)：在神经概率语言模型中系统引入可学习词向量。
> - Mikolov et al. (2013)：通过 word2vec 展示基于局部预测学习高质量词向量的高效路径。
> - Pennington, Socher, and Manning (2014)：通过 GloVe 展示全局共现统计与向量表示之间的关系。
> - Devlin et al. (2019)：以 BERT 为代表，推动上下文化 embedding 成为主流表示方式。

## 0. 符号约定与公式索引

为避免术语混乱，本文统一使用以下记号：

| 符号 | 含义 |
| --- | --- |
| \(V\) | 离散对象的集合，如词表、商品集合、用户 ID 集合 |
| \(\lvert V \rvert\) | 集合大小 |
| \(w\) | 一个离散符号，通常表示词或 token |
| \(i\) | 离散对象的索引 |
| \(e_i\in\mathbb{R}^{\lvert V\rvert}\) | 第 \(i\) 个对象的 one-hot 向量 |
| \(d\) | embedding 维度 |
| \(E\in\mathbb{R}^{\lvert V\rvert\times d}\) | embedding 矩阵，第 \(i\) 行是对象 \(i\) 的向量 |
| \(x_i\in\mathbb{R}^d\) | 对象 \(i\) 的 embedding 向量 |
| \(s(i,j)\) | 对象 \(i,j\) 的相似度函数 |
| \(\theta\) | 除 embedding 外的其余模型参数 |
| \(\mathcal{L}\) | 训练目标函数 |

本文核心公式索引：

1. one-hot 到 embedding 的查表形式：
\[
x_i = E^\top e_i
\]

2. 两个对象的点积相似度：
\[
s(i,j)=x_i^\top x_j
\]

3. 基于 softmax 的条件概率形式：
\[
P(j\mid i)=\frac{\exp(x_i^\top y_j)}{\sum_{k\in V}\exp(x_i^\top y_k)}
\]

4. 作为神经网络输入的嵌入层映射：
\[
h=f(Wx_i+b)
\]

## 1. 定义与建模假设

Embedding 的核心目标，是把原本离散、不可直接比较的符号映射到连续向量空间中，使模型可以用线性代数运算来表达“相似、相关、可替换、可组合”等关系。

它本身通常不是一个独立完成任务的模型，更准确地说，它是现代机器学习与自然语言处理中的**基础表示层**。文本中的词、子词、位置编号、类别标签、推荐系统中的用户与物品 ID，都可以先经过 embedding，再交给后续模型处理。

Embedding 相对 one-hot 表示的核心改进在于：

- **从离散编码转向连续表示**：one-hot 只表示“是不是同一个对象”，而 embedding 允许表达“像不像”；
- **从参数独立转向参数共享**：相近对象可以在向量空间中靠近，从而共享统计强度；
- **从高维稀疏转向低维稠密**：输入维度显著降低，更适合神经网络优化；
- **从符号查表转向结构表达**：向量之间的距离、方向、夹角可以编码语义或行为模式。

Embedding 隐含的建模假设通常包括：

- **分布式表示假设**：对象的属性不是由单个维度独占表示，而是分散编码在多个维度中；
- **邻近可迁移假设**：语义、功能或行为相近的对象，其向量应当彼此靠近；
- **任务驱动假设**：embedding 的几何结构不是预先定义的，而是在某个训练目标下学出来的；
- **低维压缩假设**：高维离散空间中的有效结构，可以在较低维的连续空间中近似保留。

例如，在词表 \(\{\text{猫}, \text{狗}, \text{汽车}\}\) 上，one-hot 表示只会告诉我们这三个词互不相同；而 embedding 希望学到：

- 「猫」与「狗」距离较近；
- 「猫」与「汽车」距离较远；
- 某些维度可能隐式编码“生物性”“宠物性”“交通工具性”等可迁移特征。

## 2. 核心机制

Embedding 最基本的形式，可以看作一个可学习的查表矩阵。若对象 \(i\) 的 one-hot 表示为 \(e_i\)，embedding 矩阵为 \(E\)，则其向量表示为：

\[
x_i = E^\top e_i
\]

由于 \(e_i\) 只有第 \(i\) 位为 1，这个运算本质上等价于“取出矩阵 \(E\) 的第 \(i\) 行”。因此，embedding 层在计算上不是一般意义的矩阵乘法，而是**按索引查参数表**。

### 2.1 从 one-hot 到稠密向量

设词表大小为 \(\lvert V\rvert=50000\)，若直接用 one-hot 作为输入，每个词都对应一个 50000 维稀疏向量。这样做存在两个问题：

- 维度极高，大多数位置恒为 0；
- 任意两个不同词的内积都为 0，无法表达相似性。

Embedding 通过一个 \(50000\times d\) 的矩阵把词映射到 \(d\) 维空间，通常 \(d\ll 50000\)。这样每个词都变为一个可训练的稠密向量，例如：

| 对象 | one-hot 特征 | embedding 特征 |
| --- | --- | --- |
| 「猫」 | 只有一个维度为 1 | 多个维度共同编码语义 |
| 「狗」 | 与「猫」完全正交 | 可与「猫」靠近 |
| 「汽车」 | 与前两者同样正交 | 可自动远离动物类词 |

### 2.2 相似度与几何关系

一旦对象被映射到向量空间，就可以定义相似度。最常见的形式包括：

- 点积：
\[
s(i,j)=x_i^\top x_j
\]
- 余弦相似度：
\[
\cos(i,j)=\frac{x_i^\top x_j}{\lVert x_i\rVert\lVert x_j\rVert}
\]

这些量不只是计算工具，也决定了 embedding 空间的几何含义：

- 距离近，通常表示上下文、功能或行为相似；
- 方向关系有时能反映某类可迁移模式；
- 模型最终利用的是“向量之间如何相互作用”，而不是单个词的孤立标签。

### 2.3 embedding 如何被学习

Embedding 本身没有固定训练方式。它的参数 \(E\) 往往和整个任务一起通过反向传播学习。若某个下游任务的损失为 \(\mathcal{L}\)，则 embedding 矩阵中的被访问行会根据梯度更新：

\[
E \leftarrow E - \eta \frac{\partial \mathcal{L}}{\partial E}
\]

这意味着：

- 在语言模型中，embedding 学到的是“有助于预测上下文或下一个词”的表示；
- 在分类模型中，embedding 学到的是“有助于判别标签”的表示；
- 在推荐系统中，embedding 学到的是“有助于拟合用户-物品交互”的表示。

因此，embedding 的本质不是“给对象随便配一个向量”，而是**让向量成为任务统计规律的可训练载体**。

### 2.4 静态 embedding 与上下文化 embedding

从表示方式看，embedding 大致可分为两类：

| 类型 | 表示方式 | 特点 | 代表方法 |
| --- | --- | --- | --- |
| **静态 embedding** | 一个对象对应一个固定向量 | 简单、高效，但一词多义处理较弱 | word2vec、GloVe |
| **上下文化 embedding** | 同一对象在不同上下文下生成不同向量 | 表达力更强，可区分语境 | ELMo、BERT、GPT 系列中的隐层表示 |

例如，「bank」在「river bank」与「open a bank account」中，静态 embedding 往往只有一个固定向量；上下文化 embedding 则会根据上下文生成不同表示。

## 3. 训练与推理流程

Embedding 的训练流程取决于它是作为独立表示学习目标，还是作为下游模型的一部分。

### 3.1 端到端任务中的训练流程

在多数神经网络中，embedding 层训练过程可以概括为：

1. 将离散输入索引 \(i\) 映射为向量 \(x_i\)；
2. 将 \(x_i\) 输入后续网络，得到隐藏表示或预测结果；
3. 计算任务损失 \(\mathcal{L}\)；
4. 通过反向传播把梯度传回 embedding 层；
5. 更新 embedding 向量以及其他参数 \(\theta\)。

若后续网络写为

\[
h=f(Wx_i+b)
\]

则 embedding 层的作用，是把离散 ID 转换为可被网络消费的连续输入。

### 3.2 预训练 embedding 的学习流程

若目标是先学一个通用 embedding，再迁移到下游任务，常见流程包括：

1. 从大规模语料或交互日志中构造共现或监督信号；
2. 设定训练目标，如上下文预测、共现拟合、对比学习或矩阵分解近似；
3. 优化 embedding 参数；
4. 将训练好的向量作为初始化，或直接固定后供下游模型使用。

这一流程在 NLP 中常见于 word2vec、GloVe，在推荐中常见于双塔召回与用户-物品表征学习。

### 3.3 推理阶段如何使用 embedding

推理时通常不再“解释 embedding 本身”，而是直接使用它参与下游计算，例如：

- 作为分类模型、序列模型或检索模型的输入；
- 计算向量相似度，用于近邻搜索或召回；
- 作为更深层网络中的初始表示，再进一步变换。

若系统是检索式场景，还会把 embedding 预先存入向量索引，在推理时做最近邻搜索。这说明 embedding 不仅服务训练，也直接影响部署阶段的检索效率与效果。

## 4. 一个最小推演例子

考虑一个极小词表：\(\{\text{国王}, \text{王后}, \text{男人}, \text{女人}, \text{苹果}\}\)。假设模型已经学到以下二维 embedding，仅用于说明机制：

| 词 | embedding（示意） | 直观解释 |
| --- | --- | --- |
| 国王 | \((0.9, 0.8)\) | 同时具有“皇室”与“男性”属性 |
| 王后 | \((0.9, 0.2)\) | 具有“皇室”与“女性”属性 |
| 男人 | \((0.2, 0.8)\) | 男性属性强，但无皇室属性 |
| 女人 | \((0.2, 0.2)\) | 女性属性强，但无皇室属性 |
| 苹果 | \((-0.7, 0.1)\) | 与人类、皇室语义都远离 |

这个例子中，第一维可粗略理解为“皇室相关性”，第二维可粗略理解为“男性相关性”。虽然真实模型中的维度通常没有如此直接的单义解释，但这个简化示例能展示 embedding 到底做了什么。

| 步骤 | 输入或操作 | 向量层面发生了什么 | 说明 |
| --- | --- | --- | --- |
| 1 | 输入「国王」 | 查表得到 \((0.9, 0.8)\) | 离散词被映射到连续空间 |
| 2 | 比较「国王」与「王后」 | 两者第一维接近 | 共享“皇室”特征 |
| 3 | 比较「国王」与「男人」 | 两者第二维接近 | 共享“男性”特征 |
| 4 | 比较「国王」与「苹果」 | 两维都明显偏离 | 语义关系弱，距离更远 |
| 5 | 线性组合示意 | \(x_{\text{国王}}-x_{\text{男人}}+x_{\text{女人}}\approx x_{\text{王后}}\) | 某些关系可以由向量差近似表达 |

这个推演说明了 embedding 相对 one-hot 的关键提升：

- one-hot 只能区分对象身份，不能表达对象之间的结构关系；
- embedding 可以把“相似性”“类别性”“关系性”编码进向量几何结构；
- 后续模型不必从零开始理解每个离散符号，而能直接利用这些连续特征。

## 5. 关键优势、局限与训练难点

Embedding 之所以有效，主要因为它把原本极其稀疏的离散统计问题，转化为连续空间中的参数共享问题。

它的主要优势包括：

- **缓解稀疏性**：相近对象共享表示结构，不再完全依赖精确匹配；
- **降低输入维度**：相较 one-hot，更适合大规模神经网络训练；
- **支持迁移与复用**：预训练向量可在多个任务中复用；
- **便于相似度计算**：可直接用于聚类、检索、召回与推荐；
- **兼容性强**：可作为几乎所有现代深度模型的前置表示层。

但 embedding 的局限也很明确：

- **语义依赖训练目标**：embedding 学到的“相似”未必符合人类语义直觉，而是服务于具体任务；
- **静态表示存在一词多义问题**：同一词在不同上下文中的意义会被混合；
- **长尾对象难以学好**：低频词、冷启动用户、稀有物品的向量质量常较差；
- **可解释性有限**：高维向量往往难以逐维解释；
- **可能继承数据偏差**：训练语料中的社会偏见、流行偏好会进入向量空间。

从训练角度看，embedding 常见难点包括：

- 高频对象更新过多，低频对象更新不足；
- 大词表或大规模 ID 集合会带来显著的存储与访存开销；
- 若训练目标过弱，向量可能只学到表面共现，而缺乏可迁移结构；
- 在检索或对比学习任务中，负样本构造方式会显著影响 embedding 几何形态。

工程上常见的缓解策略包括：

- 子词建模、共享参数或哈希 embedding；
- 频次截断、采样、分桶与正则化；
- 采用更适合检索的对比目标；
- 使用预训练初始化，再在下游任务中微调。

## 6. 与前后代模型的关系

从历史位置看，Embedding 不是某个单一时代的短期技巧，而是现代表示学习的基础接口。

在它之前，许多模型主要依赖：

- one-hot 编码；
- 离散计数特征；
- 手工设计的符号特征或统计特征。

这些方法的共同问题在于：特征之间缺少连续结构，泛化能力弱。Embedding 的引入，意味着模型开始默认采用“先学表示，再学决策”的路径。

在它之后，表示学习沿着两条路线继续发展：

- **静态分布式表示**：如 word2vec、GloVe，把每个词映射为固定向量；
- **上下文化表示**：如 ELMo、BERT、GPT，把 embedding 与上下文编码过程进一步耦合。

可以用下表概括其演化位置：

| 表示方式 | 是否连续 | 是否上下文化 | 参数共享能力 | 典型问题 |
| --- | --- | --- | --- | --- |
| **one-hot** | 否 | 否 | 几乎没有 | 稀疏、不可泛化 |
| **静态 embedding** | 是 | 否 | 强 | 一词多义、语境不足 |
| **上下文化 embedding** | 是 | 是 | 更强 | 计算代价更高 |

因此，Embedding 解决的不是“所有建模问题”，而是一个更基础的问题：**如何让离散对象以适合优化与泛化的方式进入模型。**

---

## 7. 小结

Embedding 的最关键思想，是把离散符号映射到低维连续向量空间，使模型能够通过向量几何关系表达相似性、相关性与组合结构。

在模型演化史上，它的重要性不在于单独构成一个完整架构，而在于它奠定了现代表示学习的基本入口。无论是 NPLM、word2vec、RNN、Transformer，还是推荐系统与多模态模型，它们之所以能高效处理大规模离散对象，几乎都建立在 embedding 这一连续表示机制之上。
