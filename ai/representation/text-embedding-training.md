# Text Embedding Training：从静态词向量到上下文化表示

本文梳理文本 embedding 的主要训练路线，重点回答“表示是如何被学习出来的”。如果想先理解 embedding 的统一定义、几何直觉与查表机制，可先阅读 [embedding.md](./embedding.md)；如果重点是向量如何进入检索系统，可继续阅读 [retrieval-embedding.md](./retrieval-embedding.md)。

---

## 一、训练问题的基本形式

文本 embedding 的训练目标，可以概括为：让离散 token、短语、句子或文档在连续空间中形成对任务有用的几何结构。这个目标既可以作为完整任务的一部分端到端学习，也可以先在大规模语料上预训练，再迁移到下游任务。

从信息来源看，常见训练信号包括：

- 上下文预测；
- 共现统计；
- 对比学习；
- 监督标签；
- 用户点击、匹配或交互日志。

因此，训练路线的差异，核心不在于“是否使用 embedding”，而在于模型让哪些对象彼此靠近、哪些对象彼此分离，以及这种几何结构是由什么损失函数塑造出来的。

---

## 二、端到端训练与预训练初始化

在多数神经网络中，embedding 层训练过程可以概括为：

1. 将离散输入索引 $i$ 映射为向量 $x_i$；
2. 将 $x_i$ 输入后续网络，得到隐藏表示或预测结果；
3. 计算任务损失 $\mathcal{L}$；
4. 通过反向传播把梯度传回 embedding 层；
5. 更新 embedding 向量以及其他参数。

若后续网络写为
$$
h=f(Wx_i+b)
$$
则 embedding 层的作用，是把离散 ID 转换为可被网络消费的连续输入。

若目标是先学一个通用 embedding，再迁移到下游任务，常见流程包括：

1. 从大规模语料或交互日志中构造训练信号；
2. 设定训练目标，如上下文预测、共现拟合或对比学习；
3. 优化 embedding 参数；
4. 将训练好的向量作为初始化，或直接固定后供下游模型使用。

这一分野决定了两类非常不同的表示学习传统：

- 一类更强调“先学通用词向量，再迁移”；
- 另一类更强调“把表示层与整个模型一起联合训练”。

## 三、Word2Vec：局部上下文预测路线

如果希望先从信息流角度理解 CBOW、Skip-gram 与训练加速机制的关系，可以先阅读下面的交互式结构图：

<Word2VecTrainingExplorer />

word2vec 的核心思想是：词的语义可以由其上下文分布来刻画。围绕这一思想，它给出了两种互为对偶的训练方案，即 CBOW 与 Skip-gram。

### CBOW：由上下文预测中心词

CBOW（continuous bag-of-words）的目标，是利用窗口内的上下文词来预测中心词。设中心位置为 $t$，上下文窗口大小为 $m$，则输入通常由
$$
w_{t-m},\dots,w_{t-1},w_{t+1},\dots,w_{t+m}
$$
构成，模型需要预测中心词 $w_t$。

一种典型写法是先对上下文向量求平均：
$$
h_t=\frac{1}{2m}\sum_{k\in\{-m,\dots,-1,1,\dots,m\}}x_{t+k}
$$
再利用 $h_t$ 去预测 $w_t$。

CBOW 的特点通常包括：

- 训练速度较快；
- 对高频模式较稳定；
- 对细粒度语义差异的表达略弱。

### Skip-gram：由中心词预测上下文

Skip-gram 则反过来，以中心词为输入，去预测其周围的上下文词。若中心词为 $w_t$，则目标是最大化窗口内多个条件概率：
$$
\prod_{-m\leq k\leq m,\;k\neq 0} P(w_{t+k}\mid w_t)
$$

与 CBOW 相比，Skip-gram 往往：

- 更适合低频词；
- 更容易学到细粒度语义关系；
- 训练代价也更高。

### 训练加速：Hierarchical Softmax 与 Negative Sampling

当词表很大时，完整 softmax 的代价会非常高。为此，word2vec 常使用两类经典加速技巧：

1. **Hierarchical Softmax**：把多分类预测改写为沿树路径的一系列二分类决策；
2. **Negative Sampling**：只保留真实词对作为正样本，再采样若干噪声词对作为负样本。

以负采样为例，常见目标可写为：
$$
\log \sigma(x_w^\top y_c)+\sum_{j=1}^{K}\log \sigma(-x_w^\top y_{n_j})
$$

这一做法的关键，不是判断词属于整个词表中的哪一类，而是学习区分“真实共现词对”与“随机拼接词对”。

## 四、GloVe 与 FastText：从全局统计到子词建模

word2vec 之外，静态词向量还沿着两条重要路线继续发展：

- GloVe 更强调全局共现统计；
- FastText 更强调子词组合。

### GloVe：把全局共现写进向量空间

GloVe（global vectors for word representation）试图把整个语料中的全局共现统计直接编码进词向量。其典型目标函数可写为：
$$
\mathcal{L}
=
\sum_{i,j} f(X_{ij})
\left(
w_i^\top \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij}
\right)^2
$$

它的代表性在于：不只依赖局部上下文预测，而是直接利用全语料的统计结构。

### FastText：把词拆成子词组合

FastText 的核心改进在于：不再把词视为不可再分的原子单位，而是把词表示为多个字符级子串向量的组合。若词 $w$ 的子词集合记为 $\mathcal{G}(w)$，则其表示可写为：
$$
x_w=\sum_{g\in\mathcal{G}(w)} z_g
$$

这种设计对 OOV、词形变化与罕见词更友好，但也会引入更高的参数与检索开销。

### 三类静态方法的对比

| 方法 | 主要建模单位 | 关键信号 | 主要优势 | 主要局限 |
| --- | --- | --- | --- | --- |
| **Word2Vec** | 整词 | 局部上下文预测 | 训练高效，表示质量均衡 | 对全局统计利用较弱 |
| **GloVe** | 整词 | 全局共现统计 | 更直接利用全语料结构 | 依赖共现矩阵构建 |
| **FastText** | 子词 + 整词 | 局部预测与子词组合 | 对 OOV 与词形变化更友好 | 仍属于静态表示 |

从历史演化看，这三类方法都属于**静态 embedding**。也就是说，同一个词无论出现在什么上下文中，最终通常都对应同一个固定向量。

## 五、从静态表示到上下文化表示

静态 embedding 的核心局限，在于一个词对应一个固定向量，无法根据上下文动态调整语义表示。为解决这一问题，表示学习逐步进入了**上下文化 embedding** 的阶段，即由整个上下文共同决定当前 token 的表示。

### ELMo：上下文化表示的开端

ELMo 不再把词表示为一个固定查表向量，而是先把整个句子送入双向语言模型，再从内部隐藏状态中提取词表示。它的重要性在于，首次系统地证明：embedding 可以不是静态参数表，而是语言模型内部随上下文变化的动态状态。

### BERT：基于 Transformer 的双向表示

BERT 借助 self-attention，让序列中的每个 token 都可以直接与句中其他 token 交互。于是，某个词的表示不再单独由该词决定，而是由整句所有词共同塑造。

从表示学习角度看，BERT 的关键价值在于：

- 强化多义词区分；
- 更好地利用长距离依赖；
- 让表示与预训练目标、上下文可见范围、网络结构共同耦合。

### GPT：自回归条件下的动态表示

GPT 一类模型通常采用从左到右的下一个 token 预测目标。它同样会产生上下文化表示，但这种表示更强地服务于单向条件生成。也正因此，BERT 与 GPT 都能产生动态 embedding，却对应不同的任务侧重点。

### 静态表示与动态表示的差异

| 特性 | 静态 embedding | 动态 embedding |
| --- | --- | --- |
| **表示来源** | 词表查表矩阵 | 上下文条件下的网络计算 |
| **是否区分语境** | 否 | 是 |
| **多义词处理** | 较弱 | 明显更强 |
| **计算开销** | 低 | 较高 |
| **典型方法** | Word2Vec、GloVe、FastText | ELMo、BERT、GPT 类模型中的隐层表示 |

因此，动态 embedding 并不是简单地“换一种词向量训练法”，而是把表示学习从对象级查表推进到序列级条件计算。

## 六、句向量与后续改进

尽管 BERT 一类模型提供了强大的上下文化 token 表示，但直接拿其隐层输出做句向量时，往往并不总是理想。一个常见问题是**各向异性**：不同句子的表示容易聚集在空间中的狭窄区域，导致余弦相似度区分度下降。

围绕这一问题，后续出现了多种改进方向，例如：

- **Sentence-BERT（SBERT）**：通过孪生网络与对比学习目标，专门优化句子级相似度表示；
- **SimCSE**：利用对比学习提升句向量空间的判别性与均匀性；
- **白化、归一化与后处理技术**：改善向量分布，使相似度计算更稳定。

这些工作说明，动态 embedding 虽然显著提升了语境表达能力，但若要把它进一步作为高质量检索或匹配表示，往往仍需要针对句子级任务进行专门设计。

---

## 七、训练路线的主线理解

若把文本 embedding 的训练史压缩为一条主线，可以概括为：

1. **静态查表表示**：解决离散符号如何进入连续空间；
2. **局部预测与全局统计**：解决如何通过语料共现学到词间关系；
3. **子词建模**：解决 OOV、词形变化与长尾词问题；
4. **上下文化表示**：解决一词多义与语境依赖问题；
5. **句子级对比学习**：解决句向量空间的判别性与可检索性问题。

这条路线说明，文本 embedding 的演化并不是不断推翻旧方法，而是在不同层面持续回答“什么样的表示更适合泛化、迁移和匹配”。

---

## 八、小结

文本 embedding 的训练路线，可以被看作从“静态参数表”走向“上下文条件表示”的过程。Word2Vec、GloVe 与 FastText 解决了静态词向量如何高效学习的问题，ELMo、BERT 与 GPT 则把表示学习进一步推进到上下文化阶段，而 SBERT、SimCSE 等工作又把句子级检索与匹配目标显式写入训练过程。

因此，训练路线专题真正回答的不是“embedding 是什么”，而是“在不同任务与时代背景下，什么样的训练目标会塑造出什么样的表示空间”。
