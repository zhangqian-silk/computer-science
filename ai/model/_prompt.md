### 模型介绍文档生成提示词（教材式结构）

```markdown
# 任务目标
角色：资深 AI 研究员。
任务：为【填入模型名称，如 LSTM、GRU、Transformer、VAE】撰写一篇**专业、清晰、偏教材式**的介绍文档。

# 总体写作要求
1. **严禁开场白**：禁止出现「今天我们来介绍……」「下面我们看看……」「接下来……」等过渡性废话。直接输出标题并进入正文。
2. **风格要求**：整体风格参考统计学习教材或模型导论，而不是研究论文综述。语言应专业、平实、可解释，不故作晦涩。
3. **结构优先**：不强制使用固定五段式，不要求机械拆成“动机 / 架构 / 优化 / 边界”五部分。应优先保证文章结构自然、连贯、易读。
4. **直觉与公式并重**：既要有最小例子、表格、直观说明，也必须包含核心数学表达。
5. **术语克制**：避免堆砌“机制型标题”“范式跃迁”“物理意义”之类空泛表达。标题和小节名要像教材目录，而不是论文题目。

# 推荐文章结构

可按下列结构生成，允许根据模型特点微调，但整体组织方式应尽量接近：

## 0. 符号约定与公式索引
- 若模型涉及多个状态量、门控量、概率量、隐变量、损失项，应先统一符号。
- 用一个表格列出核心记号及含义。
- 若模型有若干关键公式，可列出“公式索引”，方便全文引用。

## 1. 定义与建模假设
- 用简明语言说明该模型是什么、处理什么数据、解决什么问题。
- 说明它相对于前代方法的核心改进点。
- 若适合，可给出一个极简输入示例。
- 应明确指出模型的核心建模假设，例如：
	- 有限上下文假设；
	- 递归状态转移；
	- 局部连接假设；
	- 潜变量生成假设；
	- 全局注意力假设。

## 2. 核心结构或概率形式
- 若是神经网络模型，应介绍核心结构与状态更新方式；
- 若是概率模型，应介绍联合分布、条件分布、似然函数或近似推断形式；
- 必须给出该模型**最关键的数学公式**；
- 每个变量都应在上下文中被解释清楚；
- 可以按“模块拆解”的方式写，如：
	- 输入层 / 状态层 / 输出层；
	- 门控结构；
	- 编码器 / 解码器；
	- 注意力模块；
	- 转移方程 / 发射方程。

## 3. 训练与推理流程
- 说明模型训练时的典型步骤；
- 若是解析解模型，可写计数、估计、平滑、归一化；
- 若是神经网络模型，可写前向传播、损失计算、反向传播、参数更新；
- 说明推理阶段如何使用该模型；
- 若适合，可用编号列表描述流程。

## 4. 一个最小推演例子
- 这是全文最重要的解释环节之一；
- 必须构造一个尽可能小、但能体现模型优势的例子；
- 建议使用 Markdown 表格展示：
	- 输入是什么；
	- 中间状态如何变化；
	- 哪一步体现了该模型相对于前代方法的提升；
	- 输出如何产生。
- 解释重点应放在“模型到底做了什么”，而不是只罗列公式。

## 5. 关键优势、局限与训练难点
- 解释该模型为什么有效；
- 说明它在优化、表达能力、长程依赖、并行性、数据需求、可解释性等方面的优势；
- 同时指出其典型局限与失效模式；
- 若该模型有典型训练技巧或稳定性手段，也应在这里说明。

## 6. 与前后代模型的关系
- 说明它与前代模型相比解决了什么问题；
- 说明它与后续模型相比又存在哪些不足；
- 若适合，可用一个对比表格，字段可包括：
	- 记忆方式；
	- 上下文范围；
	- 并行性；
	- 复杂度；
	- 核心优缺点。

## 7. 小结
- 用一段简洁、客观的话总结该模型的本质；
- 总结要回答两个问题：
	1. 这个模型最关键的思想是什么？
	2. 它在模型演化史上的位置是什么？

# 表达细则
1. **标题风格**
	- 一级标题格式：`# 模型名：一句简洁定性`
	- 二级标题风格：`## 0. 符号约定与公式索引`、`## 1. 定义与建模假设`
	- 避免使用夸张、营销式、论文式标题。

2. **示例要求**
	- 至少给出一个极简例子；
	- 优先用文本序列、时间序列、分类样本等最小案例；
	- 示例必须服务于解释机制，不要为了举例而举例。

3. **公式要求**
	- 必须包含核心公式；
	- 公式数量适中，不堆砌；
	- 公式之后要解释“它表示什么”；
	- 不要只给符号，不给意义。

4. **表格要求**
	- 适合用表格的地方尽量用表格；
	- 表格应承担解释功能，而不是仅仅美观；
	- 可用于符号说明、步骤展开、模型对比、最小推演。

5. **语言要求**
	- 语言偏专业、学术，但不能空泛；
	- 尽量多用「是什么、为什么、怎么做」的解释方式；
	- 少用抽象大词，多用可验证的描述；
	- 不写口语化互动句，不写面向读者的煽动性表达。

6. **术语要求**
	- 优先使用论文、教材和主流技术文档中的常用术语，不随意自造“看起来像术语”的中文短语；
	- 当某一概念存在较稳定的教材或论文表述时，优先使用该标准表述；解释性改写不能替代标准术语本身；
	- 若中文译名存在多种写法，全文只保留一种，并保持前后一致；
	- 首次出现的重要术语，建议采用「中文名（英文术语）」的形式；
	- 对解释性命名，应以标准术语为主，解释性说法放在后面；
	- 典型统一示例：
		- 使用「马尔可夫性假设」，不混用「马尔科夫性假设」；
		- 使用「隐藏状态（hidden state）」，不随意写成「隐状态」；
		- 使用「长程依赖（long-range dependencies）」，不混用「长距离依赖」；
		- 使用「递归状态转移（recurrent state transition）」等常见表述，避免不稳定的自造术语。

# 输出要求
- 直接输出完整 Markdown 文档；
- 不输出提示词解释；
- 不输出“以下是生成内容”之类包裹语；
- 保持章节之间有清晰分隔；
- 若某些章节对该模型不适用，可合并或略作调整，但整体风格必须保持为**教材式、说明式、结构化**。
```
