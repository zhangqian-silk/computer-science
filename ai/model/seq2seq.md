# Seq2Seq：从输入序列到输出序列的条件生成框架

> 相关文献：
> - Sutskever, Vinyals, and Le (2014)：提出基于编码器-解码器的序列到序列学习框架。
> - Cho et al. (2014)：提出 RNN Encoder-Decoder，系统化编码器-解码器结构。
> - Bahdanau, Cho, and Bengio (2015)：引入 attention，缓解固定长度上下文向量瓶颈。
> - Vaswani et al. (2017)：以 Transformer 重构 Seq2Seq 主干，使其从递归走向 self-attention。

## 符号约定与核心公式

本文统一使用以下记号：

Seq2Seq 是一个框架而非单一论文对象。下文会同时区分：
- Sutskever, Vinyals, and Le (2014) 的固定长度上下文向量版本；
- Bahdanau, Cho, and Bengio (2015) 的 attention 版本。

| 符号 | 含义 |
| --- | --- |
| \(X=(x_1,\dots,x_n)\) | 输入序列 |
| \(Y=(y_1,\dots,y_m)\) | 输出序列 |
| \(h_t\) | 编码器在输入位置 \(t\) 的隐藏状态 |
| \(s_t\) | 解码器在输出步 \(t\) 的隐藏状态 |
| \(c\) | 原始 Seq2Seq 中由编码器最终状态构成的固定长度上下文向量 |
| \(c_t\) | 解码器在输出步 \(t\) 的上下文向量 |
| \(\alpha_{t,i}\) | attention 版本中，第 \(t\) 个解码步对第 \(i\) 个输入位置分配的权重 |
| \(P(Y\mid X)\) | 给定输入序列后的条件输出分布 |
| \(\theta\) | 模型参数 |
| \(\mathcal{L}\) | 训练损失 |
| \(y_{<t}\) | 输出步 \(t\) 之前已生成的前缀 |
| \(\hat{y}_t\) | 模型在步 \(t\) 的预测输出 |

本文核心公式索引：

1. 条件分解形式：
\[
P(Y\mid X)=\prod_{t=1}^{m}P(y_t\mid y_{<t},X)
\]

2. 编码器状态更新：
\[
h_t=f_{\mathrm{enc}}(x_t,h_{t-1})
\]

3. 原始 Seq2Seq 的解码器更新：
\[
s_t=f_{\mathrm{dec}}(y_{t-1},s_{t-1},c)
\]

4. attention-Seq2Seq 的上下文向量：
\[
c_t=\sum_{i=1}^{n}\alpha_{t,i}h_i
\]

5. attention-Seq2Seq 的输出概率：
\[
P(y_t\mid y_{<t},X)=\mathrm{softmax}(W_o[s_t;c_t]+b_o)
\]

## 模型定义与条件生成假设

Seq2Seq（Sequence-to-Sequence）不是某一种单独的网络层，而是一类**从一个输入序列映射到另一个输出序列**的条件生成框架。它适用于输入和输出长度都可能变化的任务，例如：

- 机器翻译；
- 文本摘要；
- 对话生成；
- 语音识别；
- 拼写纠错与文本改写。

它相对于普通分类模型的关键区别在于：输出不再是一个固定标签，而是一整个序列；模型需要逐步生成每一个输出 token，并让后续输出依赖前面的生成结果。

Seq2Seq 通常建立在以下假设之上：

- **条件生成假设**：输出序列应视为输入序列条件下的生成结果；
- **自回归分解假设**：整个输出概率可以拆成逐步条件概率的连乘；
- **中间表示假设**：输入序列可以被编码为某种可供解码器读取的表示；
- **对齐或依赖假设**：输出中的每一步通常依赖输入中某些局部或全局部分。

例如，把英文句子「I love you」翻译成中文「我 爱 你」，本质上不是一次性输出一个整体标签，而是按顺序生成「我」「爱」「你」，并在每一步都参考输入句子。

## 编码器—解码器结构与信息流

Seq2Seq 的经典结构由两部分组成：

- **Encoder**：读取输入序列，得到中间表示；
- **Decoder**：根据中间表示逐步生成输出序列。

### 概率分解

Seq2Seq 的核心建模目标是：

\[
P(Y\mid X)=\prod_{t=1}^{m}P(y_t\mid y_{<t},X)
\]

它表示：输出序列的概率由一系列逐步条件概率组成。也就是说，模型在第 \(t\) 步生成什么，不仅取决于输入 \(X\)，还取决于之前已经生成的输出前缀 \(y_{<t}\)。

### 编码器：把输入序列变成可读表示

在早期 RNN/LSTM 版本的 Seq2Seq 中，编码器按顺序处理输入：

\[
h_t=f_{\mathrm{enc}}(x_t,h_{t-1})
\]

其中 \(h_t\) 表示编码器在位置 \(t\) 读完输入后的状态。最简单的做法是把最后一个隐藏状态 \(h_n\) 作为整段输入的摘要向量 \(c\)，交给解码器使用。

这种设计的直观含义是：

- 输入序列被压缩成一个固定长度表示；
- 解码器从这个表示中“读出”需要的输出；
- 整个模型像“先理解，再表述”。

### 解码器：逐步生成输出序列

解码器通常以自回归方式工作。若没有 attention，可写为：

\[
s_t=f_{\mathrm{dec}}(y_{t-1},s_{t-1},c)
\]

然后根据当前状态预测下一个 token：

\[
P(y_t\mid y_{<t},X)=\mathrm{softmax}(W_o[s_t;c]+b_o)
\]

这里的关键在于：

- 解码器在每一步都读取前一步生成结果；
- 输出历史会影响后续生成；
- 模型天然适合处理变长输出。

### Attention 如何缓解固定向量瓶颈

经典 Seq2Seq 的一个重要问题是：若把整段输入压缩进单个固定向量 \(c\)，长句信息很容易丢失。Attention 机制正是为缓解这一问题而引入。

在带 attention 的 Seq2Seq 中，解码器第 \(t\) 步会对编码器所有隐藏状态进行加权读取：

\[
c_t=\sum_{i=1}^{n}\alpha_{t,i}h_i
\]

其中 \(\alpha_{t,i}\) 表示解码器在第 \(t\) 步对输入位置 \(i\) 的关注程度。于是输出概率变为：

\[
P(y_t\mid y_{<t},X)=\mathrm{softmax}(W_o[s_t;c_t]+b_o)
\]

这意味着：

- 不同输出步可以关注输入中的不同部分；
- 模型不再依赖单一固定长度摘要；
- 输入输出之间的软对齐关系得以显式建模。

### 从 RNN-Seq2Seq 到 Transformer-Seq2Seq

从结构演化看，Seq2Seq 并不等于 RNN。它更准确的理解方式是：**一种输入序列到输出序列的任务框架**。其主干网络可以不断变化：

| 版本 | 编码器/解码器主干 | 特点 |
| --- | --- | --- |
| **RNN Seq2Seq** | RNN / LSTM | 顺序建模自然，但长依赖困难 |
| **Attention Seq2Seq** | RNN / LSTM + attention | 缓解固定向量瓶颈 |
| **Transformer Seq2Seq** | self-attention + cross-attention | 并行性强，建模能力更强 |

## 训练过程与推理方式

Seq2Seq 的训练与推理都围绕“逐步生成输出序列”展开，但两者细节并不完全相同。

### 训练阶段

训练时的典型流程如下：

1. 编码器读取输入序列 \(X\)，得到隐藏表示；
2. 解码器在每一步接收上一个目标词或其 embedding；
3. 解码器结合自身状态与输入表示，预测当前输出词的分布；
4. 用真实目标词计算损失；
5. 对整段输出求和或求平均，反向传播更新参数。

训练目标通常是最大化条件对数似然，等价于最小化交叉熵损失：

\[
\mathcal{L}=-\sum_{t=1}^{m}\log P(y_t^\ast\mid y_{<t}^\ast,X)
\]

这里 \(y_t^\ast\) 表示真实目标词。

### 教师强制

Seq2Seq 训练中一个非常典型的技巧是教师强制（teacher forcing）：在训练第 \(t\) 步时，解码器输入的不是自己上一步预测出的 \(\hat{y}_{t-1}\)，而是真实词 \(y_{t-1}^\ast\)。

它的优点是：

- 训练更稳定；
- 梯度传播更直接；
- 收敛速度通常更快。

但它也带来训练-推理不一致问题：训练时模型总能看到正确历史，推理时却只能依赖自己生成的历史。

### 推理阶段

推理时没有真实目标前缀可用，模型必须自回归展开：

1. 从起始符开始；
2. 预测第一个输出词；
3. 把预测结果送回解码器；
4. 继续生成直到结束符或达到最大长度。

常见解码策略包括：

- Greedy decoding；
- Beam search；
- Top-k / Top-p sampling。

这些策略的差别，不在于模型结构，而在于如何从每一步输出分布中选择最终 token。

## 最小推演：条件生成如何展开

考虑一个极简翻译任务：

\[
X=[\text{I},\ \text{love},\ \text{you}]
\]

目标输出为：

\[
Y=[\text{我},\ \text{爱},\ \text{你}]
\]

若使用带 attention 的 Seq2Seq，其过程可概括如下：

| 步骤 | 当前状态 | 模型内部发生了什么 | 结果 |
| --- | --- | --- | --- |
| 1 | Encoder 读取 `I / love / you` | 得到隐藏状态 \(h_1,h_2,h_3\) | 输入被编码为一组可读表示 |
| 2 | Decoder 初始步 | 根据起始符与编码结果生成第一个词 | 倾向输出「我」 |
| 3 | 生成第二个词时 | attention 对 `love` 赋更高权重 | 更容易输出「爱」 |
| 4 | 生成第三个词时 | attention 对 `you` 赋更高权重 | 更容易输出「你」 |
| 5 | 输出结束符 | 解码停止 | 完成整句翻译 |

这个例子体现了 Seq2Seq 的本质：

- 编码器先把输入序列变成可供读取的内部表示；
- 解码器再按时间步逐个生成输出；
- attention 使“当前该看输入哪一部分”变成动态可学的过程。

## 优势、局限与训练要点

Seq2Seq 的主要价值，在于它把大量“输入序列 → 输出序列”的任务统一进一个条件生成框架中。

它的主要优势包括：

- **适用于变长输入与变长输出**：不要求固定长度映射；
- **统一性强**：翻译、摘要、问答、改写等任务都可归入同一框架；
- **可解释生成过程**：输出是逐步产生的，每一步都有明确条件依赖；
- **易与 attention 结合**：可显式建模输入输出对齐关系。

其局限与难点也很明显：

- **早期固定向量版存在严重信息瓶颈**；
- **自回归生成速度慢**：输出必须逐步展开；
- **误差会累积**：一旦前面生成错误，后续可能持续偏离；
- **训练-推理不一致**：教师强制使训练历史与推理历史不同。

从训练角度看，Seq2Seq 常见问题包括：

- 长序列下 RNN 版本容易梯度消失或爆炸；
- 解码器容易过度依赖局部高频模式；
- Beam search 可能偏向短句或通用句式；
- 在低资源条件下，输入输出对齐不稳定。

常见缓解手段包括：

- 使用 attention 改善长句建模；
- 使用 LSTM / GRU 提升递归稳定性；
- 采用 scheduled sampling 等策略缓解曝光偏差；
- 进一步采用 Transformer 作为更强的 Seq2Seq 主干。

## 历史位置与相关模型

Seq2Seq 的历史位置可以理解为：它把“序列建模”从单纯的下一词预测或序列分类，推进到**通用条件序列生成**。

在它之前，很多模型主要擅长：

- 给定上下文预测一个词，如 n-gram、NPLM；
- 给定整段输入输出一个标签，如文本分类模型；
- 处理固定形式的局部序列预测。

Seq2Seq 的突破在于：模型第一次以统一框架处理“输入一段序列，输出另一段序列”这类更复杂任务。它的演化关系大致如下：

| 阶段 | 核心机制 | 主要问题 | 代表意义 |
| --- | --- | --- | --- |
| **N-gram / NPLM** | 固定上下文预测 | 只能做局部条件建模 | 序列概率建模起点 |
| **RNN / LSTM** | 递归状态传递 | 主要处理单序列表示 | 为 Seq2Seq 提供主干 |
| **Seq2Seq** | 编码器-解码器 | 固定向量瓶颈 | 通用条件生成框架形成 |
| **Seq2Seq + Attention** | 动态输入读取 | 仍受递归限制 | 大幅提升翻译与摘要效果 |
| **Transformer Seq2Seq** | self-attention + cross-attention | 长序列代价高 | 成为现代条件生成主流 |

因此，Seq2Seq 不是某个已经过时的具体网络，而是一条延续至今的任务建模主线。即使今天许多系统底层已从 RNN 换成 Transformer，人们仍然会把翻译、摘要、改写等任务称为 Seq2Seq 任务。

---

## 总结

Seq2Seq 的最关键思想，是把输出序列建模为输入序列条件下的逐步生成过程，并通过编码器-解码器结构把“理解输入”和“生成输出”连接起来。

在模型演化史上，它首次把机器翻译、摘要、对话等变长条件生成任务统一到同一个框架中；attention 缓解了其固定向量瓶颈，Transformer 则进一步重构了其主干实现。今天的许多生成模型虽然结构已大幅演化，但仍然建立在 Seq2Seq 这一条件生成思想之上。
