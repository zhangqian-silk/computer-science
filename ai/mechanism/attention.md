# Attention：按相关性动态聚合信息

> 相关文献：
> - Bahdanau, Cho, and Bengio (2015)：在神经机器翻译中提出可学习对齐机制，开启现代 attention 方法。
> - Luong, Pham, and Manning (2015)：系统比较多种 attention 打分形式，推动其在 Seq2Seq 中普及。

## 0. 符号约定与公式索引

本文统一使用以下符号：

| 符号 | 含义 |
| --- | --- |
| \(q\in\mathbb{R}^d\) | query，表示“当前需要什么信息” |
| \(k_i\in\mathbb{R}^d\) | 第 \(i\) 个 key，表示“第 \(i\) 个位置提供什么信息索引” |
| \(v_i\in\mathbb{R}^{d_v}\) | 第 \(i\) 个 value，表示“第 \(i\) 个位置实际携带的信息” |
| \(K=[k_1,\dots,k_n]\) | 所有 key 的集合 |
| \(V=[v_1,\dots,v_n]\) | 所有 value 的集合 |
| \(\alpha_i\) | 对第 \(i\) 个位置的注意力权重 |
| \(s(q,k_i)\) | query 与 key 的匹配分数 |
| \(c\) | attention 输出的上下文向量 |
| \(Q,K,V\) | 矩阵形式的 query、key、value |
| \(d_k\) | key / query 的维度 |

本文核心公式索引：

1. attention 权重：
\[
\alpha_i=\frac{\exp(s(q,k_i))}{\sum_{j=1}^{n}\exp(s(q,k_j))}
\]

2. 上下文向量：
\[
c=\sum_{i=1}^{n}\alpha_i v_i
\]

3. 缩放点积 attention：
\[
\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
\]

4. 加性 attention 打分：
\[
s(q,k_i)=w^\top \tanh(W_q q + W_k k_i)
\]

## 1. 定义与建模假设

Attention 的核心思想，是在生成当前表示或输出时，不再把所有历史信息压缩进一个固定长度状态，而是**根据当前需求，从一组候选信息中动态选择并加权聚合相关部分**。

它最早在 Seq2Seq 翻译模型中流行起来，目的是缓解编码器必须用单个向量概括整句输入所带来的信息瓶颈。后来，这一思想逐渐扩展为一种更一般的计算模式：给定一个 query，在 key-value 记忆库中按相关性取信息。

Attention 通常建立在以下建模假设之上：

- **相关性选择假设**：当前决策并不需要平均使用全部历史，而应优先利用与当前 query 最相关的信息；
- **软选择假设**：信息读取可以是连续加权的，而不是非此即彼的硬检索；
- **内容寻址假设**：模型可以根据内容相似度，而不是仅靠固定位置，决定该关注哪里；
- **可微聚合假设**：读取过程应当可导，从而能够端到端训练。

例如，在机器翻译中，当解码器生成中文词「苹果」时，更应该关注英文源句中对应的 “apple”，而不是平均看所有词。Attention 做的正是这种“按当前词的需要去读源句”的事情。

## 2. 核心机制

Attention 可以抽象为一个三步过程：

1. 用 query 表示“当前要找什么”；
2. 计算 query 与各个 key 的匹配分数；
3. 用归一化后的权重对 value 做加权平均，得到上下文向量。

### 2.1 从匹配分数到加权聚合

设当前 query 为 \(q\)，候选记忆单元为 \(\{(k_i,v_i)\}_{i=1}^{n}\)。先计算匹配分数 \(s(q,k_i)\)，再经 softmax 得到注意力权重：

\[
\alpha_i=\frac{\exp(s(q,k_i))}{\sum_{j=1}^{n}\exp(s(q,k_j))}
\]

最后，对 value 做加权求和：

\[
c=\sum_{i=1}^{n}\alpha_i v_i
\]

这里的 \(c\) 就是 attention 输出。它不是从固定位置读取的，而是根据 query 动态组合出来的。

这套机制的直观含义是：

- score 决定“看谁更多”；
- softmax 决定“权重如何分配”；
- weighted sum 决定“最终读出什么内容”。

### 2.2 常见打分函数

Attention 的关键之一，是如何定义 \(s(q,k_i)\)。常见形式包括：

| 打分方式 | 公式或形式 | 特点 |
| --- | --- | --- |
| **点积 attention** | \(q^\top k_i\) | 简单高效，适合并行计算 |
| **缩放点积 attention** | \(\frac{q^\top k_i}{\sqrt{d_k}}\) | 防止维度大时分数过大 |
| **加性 attention** | \(w^\top \tanh(W_q q + W_k k_i)\) | 更灵活，早期 Seq2Seq 常用 |
| **双线性 attention** | \(q^\top W k_i\) | 在表达力与效率之间折中 |

其中，Transformer 中最常用的是缩放点积 attention：

\[
\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
\]

这里 \(QK^\top\) 一次性得到所有 query 与所有 key 的匹配分数矩阵，因此非常适合 GPU 并行。

### 2.3 query、key、value 的角色

很多初学者的困惑在于：为什么要分成 query、key、value 三个量？

可以把它理解为：

- **query**：当前任务想找什么；
- **key**：每个候选位置用什么索引来表示自己；
- **value**：每个候选位置真正提供什么内容。

这种分离的好处在于：模型可以用一套向量空间做匹配，用另一套向量空间承载输出内容，从而提高表示灵活性。

### 2.4 attention 的几种典型形态

按信息来源划分，attention 常见有以下几类：

| 类型 | query 来源 | key/value 来源 | 典型用途 |
| --- | --- | --- | --- |
| **Encoder-Decoder Attention** | 解码器当前状态 | 编码器输出 | 机器翻译、摘要 |
| **Self-Attention** | 序列自身 | 序列自身 | Transformer 编码器、解码器 |
| **Cross-Attention** | 一个模态或一段序列 | 另一模态或另一段序列 | 多模态融合、检索增强 |

其中，self-attention 最有代表性，因为它允许序列中每个位置直接与其他所有位置交互，而不必通过递归链条逐步传递信息。

### 2.5 多头 attention

单个 attention 头只能在一个表示子空间中进行匹配。多头 attention（multi-head attention）通过把表示拆成多个子空间，并行执行多次 attention，再把结果拼接起来，使模型能从不同角度读取信息。

其直观作用包括：

- 一个头关注局部搭配关系；
- 一个头关注长距离依赖；
- 一个头关注句法结构；
- 一个头关注指代或对齐关系。

虽然实际训练中头的功能未必如此整齐，但多头机制确实扩展了模型的关系建模能力。

## 3. 训练与推理流程

Attention 通常作为更大模型中的一个模块参与训练，而不是单独脱离任务存在。

### 3.1 训练阶段

在典型神经网络中，attention 的训练流程可以概括为：

1. 从输入表示中线性变换得到 query、key、value；
2. 计算 attention 分数矩阵；
3. 经 softmax 得到注意力权重；
4. 对 value 做加权求和，得到上下文表示；
5. 把上下文送入后续层，计算损失；
6. 通过反向传播更新 attention 参数与主网络参数。

若以矩阵形式表示，则核心计算是：

\[
\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
\]

训练中，模型会自动学习：

- 什么样的 query 与 key 应该互相匹配；
- 哪些 value 对当前任务更有用；
- 不同头之间如何分工。

### 3.2 推理阶段

推理时，attention 的计算方式与训练时基本一致，但在自回归生成任务中存在两个关键工程点：

- **因果掩码**：当前位置只能看自己和过去，不能看未来；
- **KV cache**：历史 key 与 value 可缓存，避免每一步重复计算全部前缀。

这使得 attention 在生成式模型中既能保留全局上下文能力，又能在工程上维持可接受的推理效率。

### 3.3 训练与推理的典型差异

| 方面 | 训练时 | 推理时 |
| --- | --- | --- |
| 可见范围 | 可看完整训练样本，解码器受 mask 约束 | 自回归场景仅能看历史 |
| 计算方式 | 常按整个序列并行 | 常逐 token 展开并配合缓存 |
| 关注重点 | 学习匹配模式与表示结构 | 提高延迟、吞吐与缓存复用效率 |

## 4. 一个最小推演例子

考虑一个简化翻译场景。英文源句为：

\[
[\text{I},\ \text{eat},\ \text{apples}]
\]

当解码器准备生成中文词「苹果」时，当前 query \(q\) 表示“我现在需要与『苹果』相关的源端信息”。编码器给出三个 key-value 对：

| 位置 | 源词 | key 的作用 | value 的作用 |
| --- | --- | --- | --- |
| 1 | I | 标识“我”相关信息 | 携带主语语义 |
| 2 | eat | 标识“吃”相关信息 | 携带动作语义 |
| 3 | apples | 标识“苹果”相关信息 | 携带宾语语义 |

若当前 query 与三个 key 的匹配分数分别为：

\[
s(q,k_1)=0.5,\quad s(q,k_2)=1.2,\quad s(q,k_3)=3.8
\]

softmax 后可能得到：

\[
\alpha_1=0.03,\quad \alpha_2=0.10,\quad \alpha_3=0.87
\]

于是上下文向量为：

\[
c=0.03v_1+0.10v_2+0.87v_3
\]

| 步骤 | 当前状态 | attention 在做什么 | 结果 |
| --- | --- | --- | --- |
| 1 | 解码器准备生成下一个词 | 形成 query \(q\) | 表示当前需要什么信息 |
| 2 | query 对齐源端各位置 | 计算与 \(k_1,k_2,k_3\) 的匹配分数 | 判断该看哪里 |
| 3 | 分数经 softmax | 获得权重 \(\alpha_1,\alpha_2,\alpha_3\) | 权重总和为 1 |
| 4 | 聚合 value | 主要读取 \(v_3\) | 上下文集中体现 “apples” 信息 |
| 5 | 生成输出 | 上下文与解码器状态共同决定下一个词 | 更容易输出「苹果」 |

这个例子说明 attention 的关键提升在于：

- 它不要求模型把整句源信息预先压进一个固定长度向量；
- 它允许解码器在每一步都重新决定“此刻最该看哪里”；
- 它把“读取记忆”这件事做成了可微、动态、内容相关的过程。

## 5. 关键优势、局限与训练难点

Attention 之所以有效，是因为它把“信息压缩”问题改造成“信息选择”问题。模型不必预先记住全部细节，而可以在需要时有选择地读取相关部分。

其主要优势包括：

- **缓解固定瓶颈**：不再依赖单一隐藏状态概括全部历史；
- **长程依赖路径短**：远距离位置可直接交互，而不必经过多步递归传递；
- **可解释性相对较强**：注意力权重常能反映模型关注位置；
- **并行性好**：尤其是 self-attention，适合矩阵化并行计算；
- **适配性强**：可用于文本、语音、图像 patch、多模态输入等多种场景。

但 attention 也有明显局限：

- **计算与显存代价高**：全局 self-attention 对长度 \(n\) 的复杂度通常为 \(O(n^2)\)；
- **注意力权重不等于严格因果解释**：高权重并不必然说明该位置在语义上“真正决定了输出”；
- **对位置机制有依赖**：若没有合适的位置编码，模型对顺序敏感性不足；
- **长上下文仍有工程瓶颈**：尽管理论上可全局交互，实践中常受显存与时延限制。

训练时常见难点包括：

- 分数过大导致 softmax 过于尖锐，训练不稳定；
- 多头之间可能出现冗余，部分头学习到相似模式；
- 长序列训练成本高，batch size 与上下文长度难以同时做大；
- 在小数据集上，attention 容易学到表面共现模式而非稳定结构。

常见缓解手段包括：

- 使用缩放点积形式控制数值范围；
- 配合残差连接、层归一化与 dropout；
- 对长序列使用局部 attention、稀疏 attention、线性 attention 或分块策略；
- 使用更合适的位置编码增强顺序建模。

## 6. 与前后代模型的关系

Attention 的历史意义，在于它改变了序列模型处理上下文的基本方式。

在 attention 普及之前，Seq2Seq 模型主要依赖编码器最后一个隐藏状态作为“整句摘要”。这种方式在短句上尚可，但句子一长，信息容易丢失。Bahdanau attention 的突破点就在于：解码器不必只看最终状态，而可以在每一步回看整个源句。

之后，attention 又从辅助模块逐步演化为主干架构：

- 在 RNN-Seq2Seq 中，它最初是对编码器状态的读取机制；
- 在 Transformer 中，它成为核心计算单元，递归结构反而退居次要；
- 在多模态模型、检索增强模型和扩散模型中，cross-attention 又成为条件注入的重要接口。

可以用下表概括其代际位置：

| 模型机制 | 上下文读取方式 | 长程依赖路径 | 并行性 | 核心问题 |
| --- | --- | --- | --- | --- |
| **N-gram** | 固定窗口查局部上下文 | 短 | 强 | 视野有限，稀疏严重 |
| **RNN / LSTM** | 递归状态传递 | 长 | 弱 | 长依赖难、训练慢 |
| **Seq2Seq + Attention** | 动态回看编码器状态 | 较短 | 中 | 显著缓解信息瓶颈 |
| **Transformer** | 全局 self-attention | 极短 | 强 | 长序列 \(O(n^2)\) 代价高 |

因此，Attention 的历史定位可以概括为：它最初是为了解决 RNN 编码压缩瓶颈而提出的对齐机制，后来演化为现代深度学习中最重要的信息交互与动态读取范式之一。

---

## 7. 小结

Attention 的最关键思想，是让模型在当前步骤根据相关性动态读取信息，而不是把所有历史压缩到一个固定状态中被动继承。

在模型演化史上，它先是修补了 Seq2Seq 的信息瓶颈，随后又在 Transformer 中上升为主干机制，深刻改变了序列建模、多模态融合与生成模型的设计方式。今天的许多核心架构，实质上都建立在 attention 这一“可微检索与加权聚合”机制之上。
