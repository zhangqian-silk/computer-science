# Positional Encoding：为序列注入位置信息

> 相关文献：
> - Vaswani et al. (2017)：在 Transformer 中提出使用位置编码补足自注意力缺乏顺序感知的问题。
> - Shaw, Uszkoreit, and Vaswani (2018)：提出相对位置表示，使注意力直接建模 token 间距离。
> - Dai et al. (2019)：在 Transformer-XL 中进一步发展相对位置编码，改进长程依赖建模。
> - Su et al. (2021)：提出 RoPE，将位置信息编码为旋转变换，广泛用于大语言模型。

## 0. 符号约定与公式索引

符号如下：

| 符号 | 含义 |
| --- | --- |
| $x_t\in\mathbb{R}^d$ | 序列中第 $t$ 个 token 的内容表示 |
| $p_t\in\mathbb{R}^d$ | 第 $t$ 个位置的编码向量 |
| $z_t\in\mathbb{R}^d$ | 注入位置信息后的输入表示 |
| $d$ | 表示维度 |
| $d_{model}$ | Transformer 原始论文中的表示维度记号 |
| $pos$ | Transformer 原始论文中的位置索引 |
| $t$ | 位置索引，通常从 0 或 1 开始 |
| $i$ | 维度索引 |
| $Q,K,V$ | 注意力中的 query、key、value 矩阵 |
| $q_t,k_t,v_t$ | 第 $t$ 个位置对应的 query、key、value 向量 |
| $d_k$ | 缩放点积注意力中 key / query 的维度 |
| $\Delta=t-s$ | 两个位置之间的相对距离 |
| $\mathrm{Attn}(\cdot)$ | 注意力计算函数 |

核心公式如下：

1. 输入表示与位置编码相加：
$$
z_t = x_t + p_t
$$

2. Transformer 原始正弦位置编码：
$$
\mathrm{PE}(pos,2i)=\sin\left(\frac{pos}{10000^{2i/d_{model}}}\right),\quad
\mathrm{PE}(pos,2i+1)=\cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)
$$

3. 缩放点积注意力：
$$
\mathrm{Attn}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

4. 带相对位置信息的打分形式：
$$
\alpha_{ts}\propto q_t^\top k_s + q_t^\top r_{t-s}
$$

## 1. 定义与直观理解

Positional Encoding 的作用，是在序列模型中显式注入“第几个位置”“两个 token 相隔多远”“谁在前谁在后”这类顺序信息。

它之所以重要，是因为自注意力机制本身对输入集合是**置换等变**的：如果只看内容向量，不加任何位置信号，那么「猫 追 狗」与「狗 追 猫」在注意力层眼中只是同一组 token 的不同排列，模型并不会天然知道顺序已经改变。

因此，位置编码所解决的核心问题不是“增加更多特征”，而是补上自注意力架构中缺失的序列归纳偏置。

这一机制可以进一步概括为以下几个核心特征：

- **顺序有信息假设**：同样一组 token，以不同顺序排列会表达不同含义；
- **位置可向量化假设**：离散位置或相对距离可以映射到连续表示空间；
- **位置与内容可组合假设**：模型可以把“是什么”与“在哪里”联合用于预测；
- **距离影响交互假设**：在许多任务中，token 间的相对距离会影响它们的相关性。

例如，句子「我 只 喜欢 你」与「只 我 喜欢 你」包含相同词集合，但语序不同。若没有位置信息，纯注意力机制难以区分这两种排列；位置编码正是用于打破这种对顺序的无感知性。

## 2. 核心机制

Positional Encoding 的基本思想，是为每个位置 $t$ 构造一个向量 $p_t$，再把它与该位置的内容表示 $x_t$ 结合，形成输入表示：

$$
z_t = x_t + p_t
$$

这样，后续层看到的就不再只是 token 的语义内容，还包含其在序列中的位置线索。

### 2.1 为什么注意力本身不够

标准自注意力的核心打分形式为：

$$
\mathrm{Attn}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d}}\right)V
$$

其中，$Q,K,V$ 都是由输入表示线性变换得到的。如果输入只含内容向量而不含位置，那么注意力打分只依赖“内容和谁相似”，而不依赖“谁在前、谁在后、相距多远”。

这意味着：

- 注意力能建模全局关联；
- 但它默认不知道局部邻接结构；
- 也不知道第一个词与最后一个词在序列中的角色差异。

位置编码就是为了解决这个结构缺口。

### 2.2 绝对位置编码

最直接的方法，是给每个绝对位置 $t$ 分配一个向量 $p_t$。这类方法可分为两种：

| 类型 | 做法 | 特点 |
| --- | --- | --- |
| **可学习绝对位置编码** | 把每个位置当作一个可训练参数 | 灵活，但长度外推能力通常较弱 |
| **固定绝对位置编码** | 由预定义函数直接生成 | 无需学习，可能更易外推 |

Transformer 原始论文采用的是固定的正弦-余弦位置编码。这里保留原论文中的 $pos$、$i$ 与 $d_{model}$ 记号，便于直接对照 Vaswani et al. (2017)：

$$
\mathrm{PE}(pos,2i)=\sin\left(\frac{pos}{10000^{2i/d_{model}}}\right),\quad
\mathrm{PE}(pos,2i+1)=\cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)
$$

它的直观含义是：

- 不同维度对应不同频率；
- 低频维度捕获较粗的位置变化；
- 高频维度捕获较细的位置变化；
- 任意位置都能由一组周期函数唯一刻画。

这种设计的一个重要优点是：位置 $t$ 不需要查表记忆，而是由解析函数直接生成，因此理论上可以推广到训练中未出现的更长序列。

### 2.3 相对位置编码

绝对位置编码回答的是“我在第几个位置”，但在注意力中，很多时候更重要的问题是“你和我相距多远”。

相对位置编码因此直接把距离 $\Delta=t-s$ 融入注意力打分。例如一种典型形式是：

$$
\alpha_{ts}\propto q_t^\top k_s + q_t^\top r_{t-s}
$$

其中 $r_{t-s}$ 表示相对距离 $t-s$ 的向量表示。这样模型在计算位置 $t$ 对位置 $s$ 的注意力时，不仅考虑内容匹配，也考虑它们的相对距离关系。

相对位置编码的优势通常包括：

- 更符合序列建模直觉，因为语言中的依赖常与距离相关；
- 对平移更稳定，例如“前一个词”“后两个词”这种模式可在不同绝对位置复用；
- 在长序列任务中，往往比绝对位置编码更稳健。

### 2.4 旋转位置编码

在大语言模型中，RoPE（Rotary Positional Embedding）是一类非常重要的位置机制。它的核心思想不是把位置向量直接加到输入上，而是对 query 和 key 的二维子空间施加与位置相关的旋转。

直观上可以把它理解为：

- 内容向量仍保留在原始空间中；
- 不同位置对应不同旋转角度；
- 两个位置之间的注意力分数，会自然带出相对位移信息。

这类方法的优点在于：位置信息不是外加偏置，而是直接进入内积结构，因此更自然地服务相对距离建模。

## 3. 在模型中的使用方式

位置编码本身通常不是独立模型，而是 Transformer 等架构中的组成部分。因此，它的使用方式需要放在整体网络中理解。

### 3.1 在不同模型中的位置

| 模型或结构 | 位置编码的使用位置 | 主要作用 |
| --- | --- | --- |
| Transformer 编码器 | 输入表示或注意力打分阶段 | 为整段输入提供顺序信息 |
| Transformer 解码器 | 自回归输入与注意力计算中 | 区分生成顺序与相对距离 |
| 大语言模型 | token 表示或 query/key 变换中 | 支持长上下文中的位置建模 |
| 多模态 Transformer | 文本 patch、图像 patch 或时间帧表示中 | 为不同模态注入排列与结构信息 |

### 3.2 训练阶段

典型流程可以概括为：

1. 将输入 token 映射为内容向量 $x_t$；
2. 为每个位置构造位置表示 $p_t$，或生成相对位置项；
3. 将内容与位置组合，得到输入表示 $z_t$；
4. 送入注意力层与前馈层，计算任务损失；
5. 通过反向传播更新模型参数，以及可学习的位置参数。

若使用固定正弦位置编码，则位置本身不更新；若使用可学习位置编码，则对应参数会像普通 embedding 一样被训练。

### 3.3 推理阶段

推理时，位置编码的使用方式与训练阶段保持一致，但要注意以下差异：

- 自回归生成中，位置索引会随生成步数递增；
- 若采用 KV cache，新增 token 只需计算新位置对应的编码，而不必重算全部历史表示；
- 长度超出训练范围时，某些绝对位置编码可能外推较差，而相对位置或旋转编码往往更稳健。

### 3.4 训练与推理中的典型差异

| 机制 | 训练时关注点 | 推理时关注点 |
| --- | --- | --- |
| 绝对位置编码 | 最大训练长度、参数规模 | 超长外推能力 |
| 相对位置编码 | 距离桶设计、注意力实现复杂度 | 长序列稳定性 |
| 旋转位置编码 | 频率设计与缩放策略 | 长上下文扩展、缓存兼容性 |

## 4. 一个最小推演例子

考虑一个极小输入序列：

$$
[\text{猫},\ \text{追},\ \text{狗}]
$$

若只使用内容向量而不加位置编码，那么模型看到的是三组 token 表示的集合，难以天然区分：

$$
[\text{猫},\ \text{追},\ \text{狗}]
\quad\text{与}\quad
[\text{狗},\ \text{追},\ \text{猫}]
$$

用绝对位置编码做一个最小推演。设内容向量分别为 $x_1,x_2,x_3$，位置编码分别为 $p_1,p_2,p_3$，则组合后的输入为：

$$
z_1=x_1+p_1,\quad z_2=x_2+p_2,\quad z_3=x_3+p_3
$$

| 步骤 | 输入或状态 | 模型内部发生了什么 | 说明 |
| --- | --- | --- | --- |
| 1 | 「猫」位于位置 1 | 得到 $z_1=x_{\text{猫}}+p_1$ | 「猫」不仅有内容，还带有“在前面”的信息 |
| 2 | 「追」位于位置 2 | 得到 $z_2=x_{\text{追}}+p_2$ | 谓词位于中间位置 |
| 3 | 「狗」位于位置 3 | 得到 $z_3=x_{\text{狗}}+p_3$ | 「狗」带有“在后面”的信息 |
| 4 | 计算注意力 | $q_t^\top k_s$ 同时受内容与位置影响 | 模型开始区分谁修饰谁、谁在前谁在后 |
| 5 | 调换顺序后重算 | 「狗 追 猫」对应 $x_{\text{狗}}+p_1$、$x_{\text{猫}}+p_3$ | 内容相同，但位置不同，注意力结果也会不同 |

这个例子说明：

- 内容向量回答“这个 token 是什么”；
- 位置编码回答“它在什么位置，和别人相隔多远”；
- 两者结合后，注意力才真正具备序列理解能力。

## 5. 常见变体、优势与局限

位置编码之所以有效，是因为它为缺乏顺序偏置的注意力结构补充了最基本的序列坐标系。

它的主要优势包括：

- **补足顺序信息**：使模型区分不同排列的同一组 token；
- **保留全局建模优势**：不必像 RNN 那样串行处理，也能感知位置；
- **设计空间丰富**：可根据任务选择绝对、相对、旋转等不同形式；
- **适配长距离依赖**：相比固定窗口方法，更容易结合全局上下文进行位置敏感建模。

其局限与难点主要包括：

- **绝对位置编码外推有限**：训练长度之外的位置可能表现不稳；
- **不同任务所需的位置归纳偏置不同**：文本、语音、图像 patch 的位置结构并不一致；
- **相对位置实现更复杂**：会改变注意力打分形式，增加工程复杂度；
- **长上下文扩展并不只靠位置编码**：还受注意力复杂度、归一化、缓存设计等因素影响。

常见失效模式包括：

- 长度远超训练分布时，模型对后段 token 的位置感知变弱；
- 位置信号过强时，模型可能过度依赖表面顺序模式；
- 位置信号过弱时，模型又会退化为“只看词、不看顺序”的内容匹配器。

工程上常见干预手段包括：

- 为长上下文任务使用相对位置编码或 RoPE；
- 采用位置插值、频率缩放等方式扩展上下文窗口；
- 在实现中配合缓存机制，避免推理时重复计算全部位置信号。

## 6. 与相关机制或模型的关系

位置编码与相关机制、相关模型之间的关系，可以放在序列建模架构的演化中理解。

在 RNN、LSTM 时代，顺序信息天然存在于递归展开过程中：模型按时间步依次读入输入，位置由处理顺序隐式给出。因此，那一代模型通常不需要专门设计位置编码。

Transformer 改用自注意力后，获得了并行计算与全局依赖建模的优势，但也失去了“顺序天然写在结构里”的特性。位置编码正是在这一背景下成为必需模块。

可以用下表概括几类序列位置信息机制的差异：

| 机制 | 顺序信息来源 | 是否显式位置编码 | 并行性 | 典型特点 |
| --- | --- | --- | --- | --- |
| **RNN / LSTM** | 递归处理顺序 | 通常不需要 | 弱 | 顺序天然存在，但难并行 |
| **Transformer + 绝对位置编码** | 外加位置向量 | 是 | 强 | 简洁直接，易实现 |
| **Transformer + 相对位置编码** | 注意力中的距离项 | 是 | 强 | 更强调相对距离关系 |
| **Transformer + RoPE** | query/key 旋转 | 是 | 强 | 与内积结构结合紧密，适合长上下文 |

因此，Positional Encoding 并不是附属技巧，而是注意力模型获得顺序建模能力时的关键补充机制。

---

## 7. 小结

Positional Encoding 的最关键思想，是为原本只擅长内容匹配的注意力机制加入位置信号，使模型能够同时理解“是什么”和“在哪里”。

在现代模型中，它通常承担顺序注入与距离建模的作用；相对位置编码与旋转位置编码则进一步把位置信息写入注意力计算本身。
