本文讨论位置机制的基本问题、主流路线与统一比较，重点回答“为什么需要位置编码、它有哪些主要形式、如何进入 attention”。若希望进一步了解 RoPE 深度实现、长度外推与超长上下文适配，可继续阅读文中的相关专题链接。

## 问题背景与核心动机

Transformer 之所以能高效训练，一个关键原因是它把序列处理改写为并行计算。模型不再像 RNN 那样按时间步依次读入「前一个、后一个」，而是把整段输入同时映射为向量，再通过 self-attention 计算位置之间的相关性。

这种并行性带来速度和表达能力上的优势，但也带来一个结构性缺口：**注意力天然擅长比较内容相似性，却不天然携带顺序信息。** 如果不给模型额外的位置信号，那么「我 吃 鱼」与「鱼 吃 我」在形式上都只是由同一组 token 构成的不同排列，模型更容易把它们视作相近的输入集合，而不是语义完全不同的句子。

位置编码的任务，就是把「顺序」重新写回表示空间。它并不只是给每个 token 附一个编号，而是在回答三个层层递进的问题：

- 这个 token 在序列中的**绝对位置**是什么；
- 两个 token 之间的**相对距离**是多少；
- 这种位置信息应当以什么形式进入 attention，才能既利于训练，又利于长上下文外推。

从学习路径看，位置编码大致可以沿三条主线理解：

- **绝对位置编码**：直接告诉模型「这是第几个位置」；
- **相对位置编码**：重点建模「两个位置相距多远」；
- **旋转位置编码（RoPE）**：把位置写入 query / key 的几何结构，使相对距离自然出现在内积中。

## 符号约定与公式索引

符号如下：

| 符号 | 含义 |
| --- | --- |
| $x_t\in\mathbb{R}^d$ | 序列中第 $t$ 个 token 的内容表示 |
| $p_t\in\mathbb{R}^d$ | 第 $t$ 个位置的编码向量 |
| $z_t\in\mathbb{R}^d$ | 注入位置信息后的输入表示 |
| $d$ | 表示维度 |
| $d_{model}$ | Transformer 原始论文中的表示维度记号 |
| $pos$ | Transformer 原始论文中的位置索引 |
| $t,s$ | 位置索引，通常从 0 或 1 开始 |
| $i$ | 维度索引 |
| $Q,K,V$ | 注意力中的 query、key、value 矩阵 |
| $q_t,k_t,v_t$ | 第 $t$ 个位置对应的 query、key、value 向量 |
| $d_k$ | 缩放点积注意力中 key / query 的维度 |
| $\Delta=t-s$ | 两个位置之间的相对距离 |
| $r_{t-s}$ | 相对距离 $t-s$ 对应的向量或偏置项 |
| $\theta_i$ | RoPE 在第 $i$ 个二维子空间上的基频 |
| $\mathrm{Attn}(\cdot)$ | 注意力计算函数 |

核心公式如下：

1. 输入表示与位置编码相加：
$$
z_t = x_t + p_t
$$

2. Transformer 原始正弦位置编码：
$$
\mathrm{PE}(pos,2i)=\sin\left(\frac{pos}{10000^{2i/d_{model}}}\right),\quad
\mathrm{PE}(pos,2i+1)=\cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)
$$

3. 缩放点积注意力：
$$
\mathrm{Attn}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

4. 带相对位置信息的打分形式：
$$
\alpha_{ts}\propto q_t^\top k_s + q_t^\top r_{t-s}
$$

5. RoPE 的二维旋转块：
$$
R_{\theta_i,t}=
\begin{bmatrix}
\cos(t\theta_i) & -\sin(t\theta_i)\\
\sin(t\theta_i) & \cos(t\theta_i)
\end{bmatrix}
$$

若只希望快速回忆不同位置机制最核心的计算形式，可以再看下面这张速查表：

| 机制 | 核心公式或形式 | 最直接表达什么 |
| --- | --- | --- |
| 绝对位置编码 | $z_t=x_t+p_t$ | 第 $t$ 个 token 的绝对坐标 |
| 正弦余弦位置编码 | $\mathrm{PE}(pos,2i),\mathrm{PE}(pos,2i+1)$ | 多频率绝对位置表示 |
| 相对位置编码 | $\mathrm{Score}(t,s)=q_t^\top k_s+b_{t-s}$ 或 $q_t^\top k_s+q_t^\top r_{t-s}$ | 两个位置之间的距离与方向 |
| RoPE | $q_t'=R_t q_t,\ k_s'=R_s k_s$ | 把位置写入 $Q,K$ 的旋转相位 |
| ALiBi | $\mathrm{Score}_{t,s}=\frac{q_t^\top k_s}{\sqrt d}-m_h(t-s)$ | 用线性距离偏置直接约束 attention |
| Position Interpolation | $m'=m\cdot\frac{L_{\text{train}}}{L_{\text{target}}}$ | 把长位置压缩回训练时熟悉区间 |

## Positional Encoding 的必要性

### 定义与直观理解

Positional Encoding 的作用，是在序列模型中显式注入「第几个位置」「两个 token 相隔多远」「谁在前谁在后」这类顺序信息。

它之所以重要，是因为 self-attention 本身对输入集合更接近**置换等变**。如果只看内容向量，不加任何位置信号，那么注意力打分主要回答的是「谁和谁语义上更相关」，而不是「谁在句首、谁在句尾、谁修饰谁」。

因此，位置编码所补上的不是普通特征，而是序列建模里非常关键的一类**归纳偏置**。这类归纳偏置至少包含以下四层含义：

- **顺序有语义含义**：同一组 token 的不同排列会表达不同意思。
- **位置可以表示为向量结构**：离散位置与相对距离都可以映射到连续空间中。
- **内容与位置需要联合建模**：模型不仅要知道「是什么」，也要知道「在哪里」。
- **距离会影响交互强度**：许多依赖关系与相对远近直接相关。

可以把这个问题理解为「词袋」与「序列」的差别。若把一句话看成无序集合，那么模型能知道句子里出现了哪些词，却很难可靠地区分主语、宾语、修饰关系与时间顺序。

---

### Self-Attention 的顺序缺口

标准 self-attention 的核心打分形式为：

$$
\mathrm{Attn}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

其中，$Q,K,V$ 都是由输入表示线性变换得到的。若输入只包含内容信息，那么注意力打分只依赖内容匹配，而不依赖顺序结构。

这意味着：

- 注意力可以建模全局关联；
- 但它默认不知道局部邻接关系；
- 也无法天然区分句首、句中、句尾这些结构角色。

换言之，self-attention 的优势是「任意位置都可以相互访问」，它的短板则是「谁在前谁在后并不写在机制里」。

## Absolute Positional Encoding（绝对位置编码）

最直接的思路，是给每个绝对位置 $t$ 分配一个向量 $p_t$。这类方法回答的问题是：「这个 token 处在整段序列的第几个位置？」

若先做一个思想实验，最直观的方案似乎是直接把位置写成整数：

$$
1,2,3,4,\dots
$$

例如句子「我 吃 鱼」可以被粗略标成「我(1) 吃(2) 鱼(3)」。但这种整数编码在神经网络里有两个明显问题：

- **数值尺度会持续增大**：当序列很长时，位置索引可能增长到几万甚至更高，容易破坏输入尺度稳定性；
- **难以形成平滑可泛化规律**：如果模型训练时只见过较短位置，直接面对更大的新整数时，往往缺少自然的外推依据。

因此，绝对位置编码的关键不只是「给出位置」，而是要找到一种**数值稳定、结构连续、可推广**的表示方式。

### Learnable Absolute Positional Encoding

最简单的写法是把位置也当作 embedding：

$$
z_t = x_t + e_t,\qquad e_t\in\mathbb{R}^d
$$

这里的 $e_t$ 与词向量一样，都是训练中学习得到的参数。它的优点是表达灵活；缺点是通常依赖最大训练长度，超出训练范围时外推能力往往有限。

| 类型 | 做法 | 优点 | 局限 |
| --- | --- | --- | --- |
| 可学习绝对位置编码 | 为每个位置维护一个可训练向量 | 灵活，易适配任务 | 长度外推通常较弱 |
| 固定绝对位置编码 | 用解析函数直接生成位置向量 | 无需额外训练位置参数 | 设计不当时表达能力可能受限 |

### Sinusoidal Encoding 的设计动机

Transformer 原始论文没有采用简单整数，也没有只用单一频率的波，而是选择了正弦与余弦函数。

第一层原因是它们的**值域天然有界**：

$$
\sin(x),\cos(x)\in[-1,1]
$$

这意味着无论位置 $pos$ 增长到多大，编码值本身始终被限制在稳定范围内，不会因为位置索引变大而出现数值爆炸。把这类编码加到词向量上时，位置信号更像一种尺度可控的坐标修正，而不是压过语义向量本身的大数输入。

但如果只用一个简单函数，例如只用 $\sin(pos)$，马上又会遇到新的问题：**周期重复**。因为

$$
\sin(0)=\sin(2\pi)=\sin(4\pi)
$$

仅靠单一频率，模型无法长期区分更远位置。也就是说，单一三角函数虽然解决了数值稳定问题，却无法独立解决「不同位置会重复撞码」的问题。

### Multi-Frequency 设计与周期分配

Transformer 原始论文采用的是固定的正弦余弦位置编码。保留原论文中的 $pos$、$i$ 与 $d_{model}$ 记号，可写为：

$$
\mathrm{PE}(pos,2i)=\sin\left(\frac{pos}{10000^{2i/d_{model}}}\right),\quad
\mathrm{PE}(pos,2i+1)=\cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)
$$

其中：

- $pos$ 表示当前位置；
- $i$ 表示第几对正弦余弦维度；
- $2i$ 与 $2i+1$ 表示偶数维和与之配对的奇数维；
- $10000^{2i/d_{model}}$ 是控制频率的缩放因子。

这里的关键并不是「偶数维用正弦、奇数维用余弦」这一表面形式，而是：**不同维度被分配了不同频率的波。**

若把位置理解为 $pos=n\cdot k+x$ 的形式，其中 $k$ 类似某个周期，那么单一频率只能告诉我们当前位置对这个周期取余后的状态，却无法唯一确定完整的绝对位置。为了避免这种歧义，Transformer 在不同维度上设置了不同的周期，就像精密时钟同时拥有秒针、分针和时针一样。

因此，某一个维度上的波形状态并不能唯一确定位置，但**所有维度上的多频率状态联合起来**，就能给出足够精细且区分性很强的位置标签。

它的直观含义是：

- 偶数维与奇数维组成一对正弦余弦坐标；
- 不同维度对应不同频率；
- 低频分量刻画较粗的位置变化；
- 高频分量刻画较细的位置变化。

因此，每个位置都由一组不同频率的周期函数共同刻画。它既不像简单位置 ID 那样过于离散，也不像纯查表那样完全依赖训练见过的位置。

从频率变化方式上看，分母

$$
10000^{2i/d_{model}}
$$

会随着维度 $i$ 增大而增大，从而让对应维度的波变化得越来越慢。低维通常更敏感于局部位置变化，高维则更像在记录更长尺度上的粗粒度位置趋势。

如果把式子改写为角频率的形式，会更直观一些：

$$
\mathrm{PE}(pos,2i)=\sin(pos\cdot \omega_i),\quad
\mathrm{PE}(pos,2i+1)=\cos(pos\cdot \omega_i)
$$

其中

$$
\omega_i = 10000^{-2i/d_{model}}
$$

这样就能看出：随着 $i$ 增大，$\omega_i$ 逐渐变小，波动速度也逐渐变慢。于是不同维度自然形成一组从高频到低频的连续频带。

### 频率基底 `10000^{2i/d_model}`

这个设计可以从三个角度理解。

第一，它需要让不同维度拥有**足够分散的频率**。如果所有维度频率太接近，那么多维位置编码仍然会高度相似，难以提供足够区分性。

第二，它需要让频率变化是**平滑连续**的，而不是突然跳变。指数形式

$$
10000^{2i/d_{model}}
$$

等价于在对数尺度上均匀铺开频率，因此当 $i$ 从 $0$ 逐渐增大到 $d_{model}/2-1$ 时，模型获得的是一组覆盖多个尺度、分布平滑的周期信号。

第三，它需要同时覆盖**短距离**与**长距离**。较小的 $i$ 对应较小的分母，因此波动更快，更容易区分相邻位置；较大的 $i$ 对应更大的分母，因此波动更慢，更适合表示更长范围内的位置趋势。

可以把这一点概括为下表：

| 维度区域 | 分母大小 | 波动速度 | 更擅长刻画什么 |
| --- | --- | --- | --- |
| 较低维 | 较小 | 较快 | 相邻位置、局部顺序 |
| 中间维 | 中等 | 中等 | 中距离位置差异 |
| 较高维 | 较大 | 较慢 | 长距离、粗粒度位置趋势 |

这可以用一个很直观的类比来理解：

- 前面的高频维度更像**秒针**，对位置变化极其敏感；
- 后面的低频维度更像**时针**，变化缓慢，但能稳定标记更长跨度的位置趋势。

因此，当维度索引 $i$ 从小到大增加时，波形会越来越平缓，而不是越来越密集。模型看到的并不是单一尺度的位置变化，而是一组从局部到全局同时展开的多尺度坐标系。

至于常数 $10000$ 本身，并不是唯一正确的神奇数字。更准确地说，它是一个经验上足够大的基数，用来保证频率覆盖范围足够宽。若换成别的基数，整体思想并不会改变，只是频率带宽与尺度分布会有所不同。

### Sin-Cos 成对设计

公式中偶数维使用 $\sin$、奇数维使用 $\cos$，本质上是在每个频率上构造一个二维坐标对。这种成对设计有两个作用：

- 它能避免只用单一函数时的信息不完整问题；
- 它能把位置变化写成二维平面上的旋转或线性变换。

如果只有 $\sin$ 而没有 $\cos$，那么模型看到的是某个投影值；加入配对的 $\cos$ 后，同一频率下的位置就能表示为一个二维向量：

$$
\begin{bmatrix}
\sin(\omega_i t)\\
\cos(\omega_i t)
\end{bmatrix}
$$

这等价于把位置 $t$ 映射到单位圆上的一个角度点。于是「位置前进一步」就会变成「这个二维向量旋转一个固定角度」，这正是后面线性位移性质成立的原因。

### Sinusoidal Encoding 的适配性

正弦余弦设计最重要的性质，不只是「能表示位置」，而是「能把位移写成线性变换」。

对任意固定频率 $\omega_i$，定义二维位置块：

$$
u_t^{(i)}=
\begin{bmatrix}
\sin(\omega_i t)\\
\cos(\omega_i t)
\end{bmatrix}
$$

则对任意偏移量 $k$，有：

$$
u_{t+k}^{(i)}=
\begin{bmatrix}
\cos(\omega_i k) & \sin(\omega_i k)\\
-\sin(\omega_i k) & \cos(\omega_i k)
\end{bmatrix}
u_t^{(i)}
$$

这说明：对于固定偏移量 $k$，位置 $t+k$ 的编码可以由位置 $t$ 的编码经过一个只依赖 $k$ 的线性变换得到。也正因此，正弦余弦编码虽然是绝对位置编码，却天然有利于模型学习相对位移模式。

这个结论并不是凭空得到的，而是直接来自三角恒等式：

$$
\sin(a+b)=\sin a\cos b+\cos a\sin b
$$

$$
\cos(a+b)=\cos a\cos b-\sin a\sin b
$$

若令 $a=\omega_i t$、$b=\omega_i k$，那么位置从 $t$ 移动到 $t+k$ 时，对应二维块的变化就能写成上面的矩阵形式。换句话说，**位移不再表现为一个全新的、离散的标签，而是表现为已有位置向量上的规则变换。**

这对模型学习有什么好处，可以从三个角度理解：

- **相同位移对应相同变换**：无论当前位置是 $t=5$ 还是 $t=105$，只要偏移量同样是 $k=1$ 或 $k=2$，其变换矩阵形式就是一致的；
- **相对距离规律更容易复用**：模型可以学会「向后看一个词」「向前看两个词」这类模式，而不必为每个绝对位置单独记忆；
- **外推更自然**：只要频率函数本身可计算，模型在更长位置上仍能生成同样形式的位置表示。

可以把这一点理解为：正弦余弦位置编码虽然表面上提供的是绝对坐标，但其内部结构天然暴露了「相对位移 = 固定线性变换」这一规律，因此它并不只是“告诉模型你在第几位”，还在暗中帮助模型学习“你和别人相差几位”。

### 最小位移示例

考虑某一对固定频率的二维位置块：

$$
u_t=
\begin{bmatrix}
\sin(\omega t)\\
\cos(\omega t)
\end{bmatrix}
$$

若当前位置从 $t$ 变到 $t+1$，则有：

$$
u_{t+1}=
\begin{bmatrix}
\cos\omega & \sin\omega\\
-\sin\omega & \cos\omega
\end{bmatrix}
u_t
$$

若再继续移动一步，从 $t+1$ 到 $t+2$，同样仍然是乘上一个只依赖于 $\omega$ 的固定矩阵。于是：

$$
u_{t+2}=A_\omega u_{t+1}=A_\omega^2 u_t
$$

其中

$$
A_\omega=
\begin{bmatrix}
\cos\omega & \sin\omega\\
-\sin\omega & \cos\omega
\end{bmatrix}
$$

这说明在同一频率子空间中，「向前移动一步」对应重复施加同一种旋转。于是：

- 偏移量 $1$ 对应一次旋转；
- 偏移量 $2$ 对应两次旋转；
- 更大的偏移量对应更多次同类变换。

把多个频率维度叠加起来后，模型看到的就是一组在不同时间尺度上同时变化的旋转坐标。也正因为如此，Transformer 虽然使用的是绝对位置编码，却并不会把每个位置都视为完全孤立的 ID。

### 从正弦余弦到 RoPE：二维旋转视角的延续

如果从几何角度回看前面的推导，会发现一个很重要的事实：正弦余弦绝对位置编码的每一对维度，本质上都在做**二维平面上的旋转表示**。

也就是说，在经典位置编码中：

- 位置 $t$ 会对应到单位圆上的一个角度；
- 位移 $k$ 会对应到这个角度上的一次额外旋转；
- 相对距离信息并不是显式写出来的，而是隐含在这些旋转关系里。

这件事非常关键，因为它解释了为什么后来的 RoPE 并不是凭空冒出来的新技巧，而更像是把这种「二维旋转编码位置」的思想，进一步推进到了 attention 内部。

二者的区别可以概括为：

| 机制 | 旋转发生在哪里 | 位置信息如何进入模型 |
| --- | --- | --- |
| 正弦余弦绝对位置编码 | 输入表示 $x_t + p_t$ 中的 $p_t$ | 先形成带位置的输入，再投影为 $Q,K,V$ |
| RoPE | query / key 的二维子空间中 | 直接在 attention 计算前旋转 $Q,K$ |

因此，两者都共享同一个核心直觉：

- 位置不是一个孤立编号；
- 位置更适合表示成多频率二维旋转坐标；
- 相对位移可以通过旋转关系自然表达出来。

只是经典正弦余弦编码把这种结构写在**输入层**，而 RoPE 把这种结构写到了**注意力的内积空间**里。

从这个角度看，可以把 RoPE 理解为一次更进一步的“结构内化”：

- 在经典正弦余弦编码里，位置信息先和 token embedding 相加，再交给后续层去学着利用；
- 在 RoPE 里，位置信息直接改写 query 和 key 的几何关系，使 attention 分数天然依赖相对位移。

这也是为什么很多现代大语言模型更偏好 RoPE：它并没有抛弃三角函数与旋转视角，反而是把它们用得更彻底了。

### Addition vs. Concatenation

当位置向量 $p_t$ 被计算出来后，Transformer 并没有把它与词向量 $x_t$ 做拼接，而是直接逐元素相加：

$$
z_t = x_t + p_t
$$

这背后有三个非常实际的原因。

第一，**维度保持不变**。如果把词向量和位置向量拼接起来，表示维度会从 $d$ 变成 $2d$，后续所有投影矩阵和 attention 层都必须跟着扩张，参数量与计算量都会显著增加。直接相加则保持模型主干维度不变，更符合 Transformer 的整体设计。

第二，**位置与内容都能同步进入每一层**。相加后的表示 $z_t$ 会同时携带语义信息和位置信息，后续线性投影生成的 $Q,K,V$ 都天然包含两者的混合结果，因此模型从第一层开始就能同时利用「是什么」和「在哪里」。

第三，**高维空间允许模型把不同类型的信息分布到不同方向上**。虽然形式上是数值相加，但在高维表示空间里，模型可以通过训练把语义信息与位置信息组织到不同子空间或不同方向上，使二者既能混合存在，又不必彼此覆盖。

因此，直接相加并不意味着位置向量会“污染”词义，而更像是给语义表示增加一个始终可见的坐标参考系。

可以把拼接与相加做一个简短比较：

| 结合方式 | 表示维度 | 工程代价 | 位置与语义的融合方式 |
| --- | --- | --- | --- |
| 拼接 | 从 $d$ 变为 $2d$ | 更高，需要改动后续层 | 先并列放置，再由后续层学习融合 |
| 相加 | 保持为 $d$ | 更低，结构最简洁 | 从输入开始就共同进入同一表示 |

也正因此，经典 Transformer 选择的是“保持维度不变的相加方案”，而不是“扩大维度的拼接方案”。

## Relative Positional Encoding（相对位置编码）

绝对位置编码回答的是「我在第几个位置」，但在 attention 中，更关键的问题往往是「你和我相距多远」。

这背后有一个很直接的观察：在很多语言现象里，真正影响依赖关系的，并不是某个词位于整段句子的第 17 个位置还是第 37 个位置，而是它和当前词之间隔了几个 token。

例如下面几类模式：

- 当前词是否需要重点关注**前一个词**；
- 某个修饰语是否出现在中心词**左侧两格**；
- 当前 token 是否和**很远之前**的主语或主题保持一致。

这些模式的共同点是，它们更像在询问「相对距离」而不是「绝对坐标」。也正因为如此，绝对位置编码虽然能提供顺序信息，但在 attention 打分里并不总是最直接的表达方式。

### Absolute Positional Encoding 的局限

设两个完全类似的局部片段分别出现在句首和句中：

$$
[\dots,\ \text{非常},\ \text{重要},\ \dots]
$$

与

$$
[\dots,\ \text{非常},\ \text{重要},\ \dots]
$$

若模型真正想学到的是「`重要` 往往会关注左边紧邻的 `非常`」，那么这个模式应当在不同绝对位置上都能复用。

但对纯绝对位置编码来说：

- 第一个片段也许对应位置 $(5,6)$；
- 第二个片段也许对应位置 $(25,26)$。

它们在绝对坐标上并不相同。模型当然仍然可以通过学习把这两种情况归纳到一起，但这种归纳是间接的；相对位置机制则直接把「左边一格」写进 attention。

因此，可以把两类机制的关注点概括为：

| 机制 | 更直接回答的问题 |
| --- | --- |
| 绝对位置编码 | 我在整段序列的第几位？ |
| 相对位置编码 | 你和我之间相隔多远？ |

### Relative Distance 与 Attention 的匹配性

attention 的核心是在计算「当前位置 $t$ 应该关注哪个位置 $s$」。既然打分本来就是成对发生的，那么把位置信息写成

$$
t-s
$$

就天然是在描述这一对位置之间的关系，而不是分别描述它们的独立绝对坐标。

这也是相对位置编码特别适合 attention 的原因：它不是先给每个位置各自打标签，再让模型自己去推导差值；而是直接把“这两个位置相距多少”写入打分函数。

从认知直觉上看，这也更符合语言理解的方式。人在阅读「猫 抓 老鼠」时，真正重要的往往不是「猫」恰好位于第 1 个词、「老鼠」恰好位于第 3 个词，而是：

- 「猫」出现在「老鼠」前面；
- 二者之间隔着一个动作词；
- 这种前后与距离关系在不同句子位置上都能重复出现。

这类模式本质上具有很强的**平移不变性**。无论整句话出现在段落开头还是结尾，「猫」与「老鼠」之间的相对关系并不会改变。

### Relative Distance 不宜预注入输入

一个很自然的想法是：既然绝对位置编码可以直接加到输入里，那么相对距离是否也能这样做？

这个想法的问题在于，**相对距离不是单个 token 的属性，而是两个 token 发生交互时才成立的关系**。

对某个单独的词 $x_t$ 来说，在进入 attention 之前，它并不知道后续会和哪一个位置 $s$ 发生比较。因此它无法提前把下面这些信息同时写进一个固定向量里：

- 我距离左边那个词多远；
- 我距离右边第三个词多远；
- 我距离句尾那个词多远。

这些关系只有在「位置 $t$ 与位置 $s$ 正在计算注意力分数」的那一刻才有明确含义。也正因为如此，相对位置信息最自然的落点不是输入层，而是**注意力打分阶段**。

相对位置编码直接把距离 $\Delta=t-s$ 写入注意力打分。典型形式之一为：

$$
\alpha_{ts}\propto q_t^\top k_s + q_t^\top r_{t-s}
$$

其中 $r_{t-s}$ 表示相对距离 $t-s$ 对应的向量或偏置项。这样一来，模型在计算位置 $t$ 对位置 $s$ 的注意力时，不仅考虑语义匹配，也考虑距离关系。

从这个公式可以看出：

- $q_t^\top k_s$ 仍然负责内容相似性；
- $q_t^\top r_{t-s}$ 则额外注入了距离结构；
- 最终的注意力强弱由「内容是否相关」与「相对位置是否合适」共同决定。

也就是说，相对位置编码并不是替代内容匹配，而是在内容匹配之外，再额外加入一条“距离偏好”的通道。

### Relative Attention 与 Relative Position Bias

在工程实现里，一个非常常见的思路是：保留原始的内容打分

$$
q_t^\top k_s
$$

然后再额外加上一项只与相对距离有关的偏置：

$$
\mathrm{Score}(t,s)=q_t^\top k_s + b_{t-s}
$$

这里的 $b_{t-s}$ 可以理解为「距离为 $t-s$ 时，应额外鼓励还是抑制多少注意力」。这类写法常被称为一种相对注意力形式，在 T5 等模型中也有非常典型的应用。

它的优点在于非常直接：

- 内容项仍然负责语义相关性；
- 偏置项专门负责距离偏好；
- 两者职责清晰，且实现上较轻量。

如果进一步使用**距离分桶**，那么模型甚至不必为每一个精确距离都维护独立参数，而只需维护若干距离区间对应的偏置即可。

### 最小直觉示例

假设当前位置是动词「吃」，模型正在决定应该关注哪个词：

$$
[\text{我},\ \text{昨天},\ \text{吃},\ \text{了},\ \text{鱼}]
$$

对于位置「吃」来说：

- 「我」与它的距离较远，但可能承担主语角色；
- 「了」与它距离很近，可能提供时态或结构信息；
- 「鱼」与它距离也较近，可能承担宾语角色。

如果模型使用相对位置编码，那么它不仅会学习「谁和我语义相关」，还会学习：

- 某些依赖常出现在左侧近距离；
- 某些依赖常出现在右侧近距离；
- 更远距离的信息虽然仍可访问，但其位置偏好模式不同。

这类信息若只靠绝对位置去学，需要模型自己把多个绝对坐标情形归纳成统一规律；相对位置编码则直接把这种规律写进结构里。

若采用偏置形式来理解，那么「吃」在和「我」「了」「鱼」分别计算分数时，模型会查找：

- 左侧较远距离对应的偏置；
- 右侧近距离对应的偏置；
- 右侧另一个近距离对应的偏置。

于是，即使这些词的内容相似度相近，最终注意力分数仍可能因为距离偏置不同而被拉开。

### 常见实现形式的差异

从实现上看，相对位置机制常见有三种写法：

| 写法 | 核心思路 | 代表性特点 |
| --- | --- | --- |
| 相对位置向量 | 为每个距离分配一个向量 $r_{t-s}$ | 直接进入打分或 value 聚合 |
| 相对位置偏置 | 为距离分配标量 bias | 实现简单，常用于编码器或 T5 类结构 |
| 距离分桶 | 把大量距离映射到少量 bucket | 降低长序列参数与实现成本 |

这三类方案可以进一步这样理解：

- **相对位置向量**更细致，允许不同距离拥有更丰富的表示能力；
- **相对位置偏置**更轻量，本质是在注意力分数上加一个和距离有关的修正项；
- **距离分桶**则是在长序列场景下做折中，让模型精细区分近距离，而把很远距离合并成较粗的类别。

例如，距离分桶常会采用这样的思路：

- 对很近的距离，精确区分 $1,2,3,4$；
- 对更远的距离，只区分「5 到 8」「9 到 16」「16 以上」这类区间。

这样做的直觉是：模型通常更需要精确知道近邻结构，而超远距离信息往往不必保持同样精细的分辨率。

### Relative Positional Encoding 在长序列中的优势

长序列中的一个难点在于：随着绝对位置不断增大，模型若过度依赖精确坐标，容易把很多规律绑定到训练中见过的特定长度范围。

相对位置机制则天然更强调「关系」而不是「坐标」，因此在以下场景里通常更稳健：

- 训练时和测试时的绝对位置分布不同；
- 同一局部结构会在序列不同位置反复出现；
- 模型更关心距离模式，而不是全局编号。

相对位置编码的优势通常包括：

- 更符合序列建模直觉，因为语言依赖往往与距离相关；
- 对平移更稳定，同一局部模式可在不同绝对位置复用；
- 在长序列任务中，往往比纯绝对位置机制更稳健。

## Rotary Positional Embedding（RoPE）

RoPE 是现代大语言模型中最重要的位置机制之一。它的核心不是把位置向量加到输入上，而是对 query / key 施加与位置相关的二维旋转，使相对位移在内积中自然出现。

若只抓住主结论，可以记住：

- 编码阶段使用绝对位置旋转；
- 打分阶段自动呈现相对距离效应；
- 工程上与自回归推理、KV cache 兼容性很好。

若希望进一步了解 RoPE 的完整数学展开、复数实现、KV cache 友好性与几何直觉，可继续阅读：

- [RoPE：把位置信息写入 Query / Key 旋转相位](./rope.md)

这里保留一个最小公式回顾：

$$
\tilde{q}_t^{(i)} = R_{\theta_i,t} q_t^{(i)},\qquad
\tilde{k}_s^{(i)} = R_{\theta_i,s} k_s^{(i)}
$$

以及它最关键的性质：

$$
\langle R_{\theta_i,t} q,\ R_{\theta_i,s} k\rangle
=
\langle q,\ R_{\theta_i,s-t} k\rangle
$$

也就是说，RoPE 虽然形式上使用绝对位置旋转，但 attention 打分里真正显现出来的是相对位移。

### RoPE 在位置机制中的角色

可以把 RoPE 理解为位置机制演化链上的第三步：

- 绝对位置编码：先写坐标；
- 相对位置编码：直接写距离；
- RoPE：把绝对旋转写进单个向量，再让相对距离在内积中显现。

因此，RoPE 既不是简单绝对位置编码，也不是普通相对位置偏置，而是一种把两者连接起来的结构化方案。

## Length Extrapolation 与 Long Context Extension

长度外推讨论的重点，不再是“RoPE 本身如何表达位置”，而是“训练长度有限时，如何把位置机制稳定延伸到更长上下文”。

这类问题的共同背景是：

$$
L_{\text{test}} > L_{\text{train}}
$$

也就是说，模型推理时需要处理超出训练长度分布的输入。对 RoPE 路线而言，这通常会带来新的相位分布与注意力模式漂移，因此发展出一系列插值、缩放与频率分层方法。

若希望系统比较这些长上下文路线，可继续阅读：

- [长上下文位置扩展：从 Length Extrapolation 到 ALiBi](./long-context-position.md)

这里仅保留最小结论：

- Position Interpolation 通过位置压缩快速扩窗；
- NTK-aware、YaRN、LongRoPE 等方法进一步做频率分层与更细粒度校准；
- ALiBi 则提供了另一条不依赖旋转相位的长上下文路线。

## ALiBi（Attention with Linear Biases）

### 基本思想与公式

ALiBi（Attention with Linear Biases）代表了一条非常不同的位置建模路线。它既不把位置向量加到输入表示里，也不对 query / key 做旋转，而是把位置信息直接写进 attention score 本身。

若以自回归注意力为例，设当前位置为 $t$、可被关注的历史位置为 $s$，其中 $s\le t$。那么 ALiBi 可以写成：

$$
\mathrm{Score}_{t,s}^{(h)}
=
\frac{q_t^{(h)\top}k_s^{(h)}}{\sqrt{d_k}}
+ m_h(s-t)
$$

由于在自回归场景下总有 $s-t\le 0$，而斜率 $m_h>0$，因此这个偏置项天然是非正的。距离越远，偏置越负，对应的扣分越大。若把它写成更直观的等价形式，也可以记作

$$
\mathrm{Score}_{t,s}^{(h)}
=
\frac{q_t^{(h)\top}k_s^{(h)}}{\sqrt{d_k}}
-m_h(t-s)
$$

这说明 ALiBi 的核心并不是“编码一个位置向量”，而是直接向模型注入一条距离先验：**越远的位置，默认越不容易获得高注意力分数。**

从 softmax 之后的效果看，这种线性扣分会转化成概率上的指数型衰减。因为

$$
\exp\left(
\frac{q_t^\top k_s}{\sqrt{d_k}}-m_h(t-s)
\right)
=
\exp\left(\frac{q_t^\top k_s}{\sqrt{d_k}}\right)\cdot \exp(-m_h(t-s))
$$

所以你若从最终注意力概率的角度观察，会发现距离项确实表现为一种指数衰减；但在 logits 层面，ALiBi 真正采用的是最简单的**线性偏置**，这也是它名称中 Linear 的来源。

ALiBi 的另一个关键设计，是在多头注意力中为不同注意力头分配不同的斜率 $m_h$。这意味着不同头会自动承担不同尺度的观察职责：

| 注意力头类型 | 斜率 $m_h$ | 偏置衰减特征 | 更擅长关注什么 |
| --- | --- | --- | --- |
| 大斜率头 | 较大 | 稍远距离就会被明显扣分 | 非常局部的近邻模式 |
| 小斜率头 | 较小 | 长距离位置仍可保留一定分数 | 更长程的跨段依赖 |

从这个角度看，ALiBi 虽然形式极简，但并没有放弃多尺度建模。它只是把“多尺度位置感”从 RoPE 的频率维度，转移到了多头注意力的不同斜率上。

ALiBi 之所以在长上下文讨论中经常出现，主要有三点原因：

- 它不需要额外构造位置向量，也不需要旋转 $Q,K$；
- 它对超出训练长度的输入可以直接沿用同一条线性规则；
- 它的工程实现非常轻量，只需在 attention score 上加一个距离偏置矩阵。

从长度外推的角度看，ALiBi 的一个天然优势是：它并不会像 RoPE 那样在超长位置上遇到“从未见过的新相位”。若训练时模型见过的最大距离是 4000，而推理时遇到距离 16000，那么对 ALiBi 来说，这并不是一个全新的几何区域，只是同一条线性规则继续向前延伸而已。若把两个距离 $d$ 与 $d+1$ 对应的偏置写出来，则有

$$
-m_h d
\qquad\text{与}\qquad
-m_h(d+1)
$$

二者只相差一个固定的 $m_h$。这意味着模型面对更长距离时，不会因为“角度超纲”而突然进入陌生分布，而只是沿着原有规则继续得到更小的 attention 倾向。因此，ALiBi 往往能表现出较强的零样本（zero-shot）长度外推能力。

但它的代价也很明确：

- 它表达的是一种很强的单调距离先验，而不是更丰富的几何相位结构；
- 若斜率设置过大，模型会过度偏向局部上下文；
- 若斜率设置过小，长距离虽保住了，但局部归纳偏置又会变弱。

因此，ALiBi 可以理解为一种“以极简线性先验换取长度外推平滑性”的方案。它不是用更复杂的数学去恢复长上下文，而是用更少的结构假设，直接规定 attention 应如何随距离衰减。

### ALiBi 的长度外推特性

若把 ALiBi 与 RoPE 并排比较，一个最核心的区别在于：**RoPE 改写的是向量的相位结构，而 ALiBi 改写的是 attention score 上的距离偏置。**

对 RoPE 而言，位置 $m$ 会直接进入旋转角 $m\theta_i$。训练时模型熟悉的是一段有限相位范围内的几何关系；一旦推理长度远超训练长度，新的 query / key 会落入更远的旋转区间，导致：

- 各维度上的相位组合进入训练中未充分覆盖的区域；
- 高频维度的局部角度差被放大或扭曲；
- 原本学到的注意力模式不再稳定。

因此，RoPE 在扩展上下文时，常需要通过插值、频率缩放或继续训练，把新的位置重新映射回模型较熟悉的相位分布中。

而 ALiBi 的距离项则更直接。它只要求对距离 $(t-s)$ 施加线性偏置：

$$
-m_h(t-s)
$$

当序列变长时，模型虽然会见到更大的距离值，但这个规则本身并没有改变，也不会像 RoPE 那样引入新的旋转相位结构。因此，ALiBi 的“外推”更像是把同一条直线继续延长，而不是把模型带入新的几何区域。

从长度外推角度看，ALiBi 常被认为具有以下天然优势：

- 距离项形式简单，不依赖训练时见过的最大绝对位置编号；
- 推理时即使序列更长，线性距离偏置仍可以按同一规则继续计算；
- 它没有“相位超纲”问题，因此不会出现 RoPE 那类由旋转分布漂移带来的失稳来源。

但这并不意味着 ALiBi 一定优于 RoPE。更准确地说，二者的取舍点不同：

| 方案 | 位置进入 attention 的方式 | 长度外推上的典型优势 | 典型代价 |
| --- | --- | --- | --- |
| RoPE | 通过旋转写入 $Q,K$ 的几何关系 | 与内积结构耦合紧，局部与相对位移表达更自然 | 超长时常需要插值、缩放或继续训练来稳住相位 |
| ALiBi | 直接向打分加入线性距离偏置 | 规则简单，超出训练长度时通常更平滑 | 表达能力更依赖偏置形式，本身不提供 RoPE 式旋转几何结构 |

可以把两者的差别进一步压缩成下面这张表：

| 方面 | RoPE | ALiBi |
| --- | --- | --- |
| 位置进入方式 | 写入 $Q,K$ 的旋转相位 | 写入 attention score 的线性偏置 |
| 超长时主要风险 | 相位分布偏离训练区间 | 距离惩罚过强或表达过于线性 |
| 常见扩展手段 | 插值、频率缩放、再校准、继续训练 | 直接外推，或微调偏置斜率设计 |
| 典型取舍 | 表达更强，但长上下文校准更重要 | 规则更简单，但几何表达较弱 |

因此，不应把“ALiBi 更容易外推”简单理解为“它绝对更好”，更准确的表述是：它通过牺牲一部分位置表达结构，换来了更直接、更稳定的长度延展性；而 RoPE 则保留了更强的几何表达能力，但在超长场景下通常需要更精细的工程校准。

### ALiBi 与一般相对位置偏置的区别

ALiBi 很容易和一般的“相对位置偏置（relative position bias）”混在一起，因为它们都直接修改 attention score，而不是像绝对位置编码那样先去改输入向量。

但二者并不完全相同。若以 T5 一类相对位置偏置为例，常见写法是：

$$
\mathrm{Score}_{t,s}
=
\frac{q_t^\top k_s}{\sqrt{d_k}}
+ b_{\mathrm{bucket}(t-s)}
$$

这里的关键是：

- 偏置值来自一个按距离 bucket 查表得到的可学习参数；
- 不同距离区间可以拥有彼此独立的偏置；
- 模型学习到的是“哪种距离更该加分或减分”，而不是一条固定直线。

相比之下，ALiBi 的形式是：

$$
\mathrm{Score}_{t,s}^{(h)}
=
\frac{q_t^{(h)\top}k_s^{(h)}}{\sqrt{d_k}}
-m_h(t-s)
$$

它不查表，也不为每个距离单独学习一个 bias，而是直接规定一条带斜率的线性规则。二者可以对比为：

| 方面 | 相对位置偏置（如 T5 bias） | ALiBi |
| --- | --- | --- |
| 距离项来源 | 查 bucket 或查表得到的可学习偏置 | 固定形式的线性函数 |
| 表达能力 | 更灵活，可对不同距离学习不同偏好 | 更简单，显式表达“越远越扣分” |
| 外推方式 | 超长距离常依赖 bucket 设计是否合理 | 可直接沿线性规则继续延伸 |
| 工程代价 | 需要距离分桶或额外偏置参数 | 实现更轻量 |

因此，ALiBi 可以被视为“相对位置偏置家族”中的一个极简特例：它保留了“直接改 attention score”这一思想，但把原本可学习、可分桶的距离偏置，压缩成了一条固定斜率的线性距离先验。

## Positional Mechanisms 对比示例

考虑两个由同样 token 组成、但语义完全不同的序列：

$$
[\text{我},\ \text{吃},\ \text{鱼}]
\quad\text{与}\quad
[\text{鱼},\ \text{吃},\ \text{我}]
$$

若只使用内容向量而不加位置编码，那么模型更容易把它们视作近似相同的 token 集合。为了更直观地比较不同机制的作用，下面用同一个小例子，并排看三类位置机制如何介入 attention。

### 无位置编码时的表现

在不加入任何位置信号时，模型看到的主要是三组内容向量：

$$
x_{\text{我}},\ x_{\text{吃}},\ x_{\text{鱼}}
$$

这时 attention 更容易回答的是：

- 「谁和谁语义相关」；
- 「哪个词更像主语、动词或宾语」。

但它很难可靠回答：

- 谁在前、谁在后；
- 哪个词处于句首或句尾；
- 两个词之间隔了多远。

因此，「我 吃 鱼」与「鱼 吃 我」虽然排列不同，却会共享几乎同样的内容集合。

### 三类位置机制对比

设当前位置为中间的动词「吃」，模型需要决定应如何关注左右两个名词。下面比较三类机制的注入方式：

| 机制 | 位置如何进入计算 | 在这个例子里模型更容易学到什么 |
| --- | --- | --- |
| 绝对位置编码 | 先构造 $z_t=x_t+p_t$，再由 $z_t$ 投影出 $Q,K,V$ | 「我」在位置 1、「鱼」在位置 3，因此句首与句尾角色不同 |
| 相对位置编码 | 在打分里直接加入 $t-s$ 的距离项 | 「吃」到左边名词和右边名词的距离、方向不同 |
| RoPE | 对 $Q,K$ 做位置相关旋转，再用旋转后向量做内积 | 左右名词与「吃」的相对位移会自然反映在 attention 分数中 |

可以把三者的差别理解为：

- 绝对位置编码更像先给每个 token 打上「你在第几位」的标签；
- 相对位置编码更像直接告诉模型「你和我相差几位」；
- RoPE 则像让向量自己带着位置相位去参与内积比较。

### 绝对位置编码下的最小推演

设内容向量分别为 $x_1,x_2,x_3$，位置编码分别为 $p_1,p_2,p_3$，则组合后的输入为：

$$
z_1=x_1+p_1,\quad z_2=x_2+p_2,\quad z_3=x_3+p_3
$$

| 步骤 | 输入或状态 | 模型内部发生了什么 | 说明 |
| --- | --- | --- | --- |
| 1 | 「我」位于位置 1 | 得到 $z_1=x_{\text{我}}+p_1$ | 「我」既有语义内容，也带有句首位置信息 |
| 2 | 「吃」位于位置 2 | 得到 $z_2=x_{\text{吃}}+p_2$ | 谓词位于中间位置 |
| 3 | 「鱼」位于位置 3 | 得到 $z_3=x_{\text{鱼}}+p_3$ | 「鱼」带有句尾位置信息 |
| 4 | 计算注意力 | $q_t^\top k_s$ 同时受内容与位置影响 | 模型开始区分主语、谓语、宾语的顺序角色 |
| 5 | 调换顺序后重算 | 「鱼 吃 我」对应 $x_{\text{鱼}}+p_1$、$x_{\text{我}}+p_3$ | 内容集合相同，但位置不同，注意力结果也会不同 |

### 相对位置编码下的最小推演

若使用相对位置编码，模型不再主要关心「我」处于第 1 位、「鱼」处于第 3 位，而更关心它们和「吃」之间的距离关系。

以位置 2 的「吃」为中心，可得到：

| 被关注词 | 相对距离 $\Delta=t-s$ | 可能提供的结构线索 |
| --- | --- | --- |
| 「我」 | $2-1=1$ 或按相反记号记为 $-1$ | 左侧近距离名词，可能是主语 |
| 「鱼」 | $2-3=-1$ 或按相反记号记为 $1$ | 右侧近距离名词，可能是宾语 |

这里模型更容易学到的不是「位置 1 常常怎样」「位置 3 常常怎样」，而是：

- 左边一格的名词与右边一格的名词扮演的角色可能不同；
- 同样的近距离关系可以在序列不同位置重复出现；
- 句法模式更像距离模式，而不是固定坐标模式。

### RoPE 下的最小推演

若使用 RoPE，位置不再通过额外向量相加，也不通过显式距离项直接加到分数上，而是通过旋转后的 query / key 内积体现出来。

在这个小例子里，可以把它理解为：

- 「吃」对应的 query 被旋转到位置 2 的相位；
- 「我」和「鱼」对应的 key 分别被旋转到位置 1 与位置 3 的相位；
- attention 比较的不是原始内容向量，而是带相位后的几何关系。

于是，模型会自然区分：

- query(位置 2) 与 key(位置 1) 的角度差；
- query(位置 2) 与 key(位置 3) 的角度差。

虽然「我」和「鱼」都可能是名词，但它们与「吃」之间的相对位移不同，因此旋转后的内积结果也会不同。

### 示例小结

这个最小例子表明：

- 内容向量回答「这个 token 是什么」；
- 位置机制回答「它在哪里」以及「它和别人是什么位置关系」；
- 三类位置机制虽然写入方式不同，但目标一致，都是让 attention 真正具备序列理解能力。

如果进一步压缩成一句话，那么三类方案的差别可以记成：

- **绝对位置编码**：先标坐标，再让模型自己利用；
- **相对位置编码**：直接写距离，再让模型按距离打分；
- **RoPE**：把位置变成旋转，再让内积自动显出相对位移。

## 工程实现、训练推理与应用场景

位置编码本身通常不是独立模型，而是 Transformer 等架构中的组成部分。因此，它的使用方式需要放在整体网络中理解。

### 在不同模型中的位置

| 模型或结构 | 位置编码的使用位置 | 主要作用 |
| --- | --- | --- |
| Transformer 编码器 | 输入表示或注意力打分阶段 | 为整段输入提供顺序信息 |
| Transformer 解码器 | 自回归输入与注意力计算中 | 区分生成顺序与相对距离 |
| 大语言模型 | token 表示或 query / key 变换中 | 支持长上下文中的位置建模 |
| 多模态 Transformer | 文本、图像 patch 或时间帧表示中 | 为不同模态注入排列与结构信息 |

### 训练阶段

典型流程可以概括为：

1. 将输入 token 映射为内容向量 $x_t$；
2. 为每个位置构造位置表示 $p_t$，或在注意力中生成相对位置项；
3. 将内容与位置组合，得到输入表示 $z_t$；
4. 送入注意力层与前馈层，计算任务损失；
5. 通过反向传播更新模型参数，以及可学习的位置参数。

若使用固定正弦位置编码，则位置本身不更新；若使用可学习位置编码，则对应参数会像普通 embedding 一样被训练。

### 推理阶段

推理时，位置编码的使用方式与训练阶段保持一致，但要注意以下差异：

- 自回归生成中，位置索引会随生成步数递增；
- 若采用 KV cache，新增 token 只需计算新位置对应的编码，而不必重算全部历史表示；
- 长度超出训练范围时，某些绝对位置编码可能外推较差，而相对位置或 RoPE 往往更稳健。

### 训练与推理中的典型关注点

| 机制 | 训练时关注点 | 推理时关注点 |
| --- | --- | --- |
| 绝对位置编码 | 最大训练长度、参数规模 | 超长外推能力 |
| 相对位置编码 | 距离桶设计、注意力实现复杂度 | 长序列稳定性 |
| 旋转位置编码 | 频率设计与缩放策略 | 长上下文扩展、缓存兼容性 |

### 与序列模型演化的关系

位置编码与相关机制、相关模型之间的关系，可以放在序列建模架构的演化中理解。

在 RNN、LSTM 时代，顺序信息天然存在于递归展开过程中：模型按时间步依次读入输入，位置由处理顺序隐式给出。因此，那一代模型通常不需要专门设计位置编码。

Transformer 改用 self-attention 后，获得了并行计算与全局依赖建模的优势，但也失去了「顺序天然写在结构里」的特性。位置编码正是在这一背景下成为必需模块。

可以用下表概括几类序列位置信息机制的差异：

| 机制 | 顺序信息来源 | 是否显式位置编码 | 并行性 | 典型特点 |
| --- | --- | --- | --- | --- |
| **RNN / LSTM** | 递归处理顺序 | 通常不需要 | 弱 | 顺序天然存在，但难并行 |
| **Transformer + 绝对位置编码** | 外加位置向量 | 是 | 强 | 简洁直接，易实现 |
| **Transformer + 相对位置编码** | 注意力中的距离项 | 是 | 强 | 更强调相对距离关系 |
| **Transformer + RoPE** | query / key 旋转 | 是 | 强 | 与内积结构结合紧密，适合长上下文 |

### 与不同数据结构的关系

位置编码虽然最常在文本 Transformer 中被讨论，但它并不只服务于一维词序列。更一般地说，只要输入被表示成一组可并行处理的 token、patch 或时间片，模型就需要某种方式回答“这些单元彼此如何排列”。

不同数据结构对“位置”的需求并不相同：

| 数据类型 | 位置结构特点 | 常见位置处理思路 | 主要关注点 |
| --- | --- | --- | --- |
| 文本序列 | 以一维先后顺序为主 | 绝对位置、相对位置、RoPE | 谁在前谁在后、词间距离、长程依赖 |
| 图像 patch | 同时具有水平与垂直二维结构 | 二维绝对位置、二维相对偏置、二维旋转扩展 | 左右、上下、局部邻域与空间布局 |
| 语音 / 时间序列 | 连续时间轴上具有局部平滑性 | 一维位置编码、相对距离、时间步偏置 | 近邻连续性、节奏、周期与长时间跨度 |
| 多模态输入 | 不同模态各自带有不同坐标系 | 模态内位置编码 + 跨模态对齐策略 | 模态内部顺序与模态之间对应关系 |

这说明位置编码并不是一个单一固定公式，而更像一类“给 attention 补充结构坐标系”的方法族。文本中的“前后顺序”、图像中的“上下左右”、时间序列中的“先后与周期”，本质上都属于不同形式的结构位置信息。

因此，当模型从纯文本扩展到视觉、语音或多模态场景时，位置机制通常也会随之变化：

- 一维序列往往更适合讨论“绝对位置”与“相对距离”；
- 二维输入往往需要把位置拆成横向与纵向两个方向；
- 多模态模型除了处理各自模态内部位置，还要处理模态之间的对齐关系。

### 现代开源 LLM 偏好 RoPE 的原因

如果把现代开源大语言模型的位置机制选择压缩成一句话，那么 RoPE 的优势在于：**它在表达能力、长度扩展潜力与系统实现友好性之间取得了非常均衡的折中。**

与前文“为什么 RoPE 在大语言模型里很常见”一节相呼应，这里更强调它成为主流默认方案的系统原因：

- 它不像传统可学习绝对位置编码那样依赖一个固定长度的位置表，位置可以在运行时按公式动态计算；
- 它把位置信息直接写入 query / key 的几何关系里，因此 attention 内积天然就能反映相对位移；
- 它与自回归推理和 KV cache 非常契合，历史 key 一旦写入缓存，就不必因为新 token 的到来而重新编码旧位置；
- 它在长上下文工程中还有进一步扩展空间，可以通过位置插值、频率缩放、YaRN、LongRoPE 等手段继续向外推长；
- 它的工程实现足够轻量，不需要像某些显式相对位置方案那样持续维护复杂的距离查表或大规模偏置结构。

若与传统绝对位置编码作一个压缩式对比，可以概括为：

| 方面 | 传统绝对位置编码 | RoPE |
| --- | --- | --- |
| 位置生成方式 | 常依赖预定义位置向量或位置表 | 运行时按位置公式动态计算 |
| 与 attention 的关系 | 先加到输入，再由后续层间接利用 | 直接写入 $Q,K$ 几何结构 |
| KV cache 友好性 | 视实现而定，长上下文扩展不总是自然 | 很友好，历史 key 可直接复用 |
| 超长上下文扩展 | 常受训练长度与参数化形式限制 | 可配合多种缩放与插值策略继续扩展 |

因此，RoPE 并不是因为在每个维度上都绝对最优才成为主流，而是因为它同时满足了现代大语言模型最看重的几项现实约束：可扩展、缓存友好、实现简洁，且与长上下文工程路线高度兼容。

从作用分工上看，还应区分三个相邻但不同的概念：

- **词向量**回答「这个 token 是什么」；
- **位置编码**回答「这个 token 在哪里」；
- **attention**回答「当前位置应该关注谁」。

三者共同组成 Transformer 中最基础的输入与交互框架。

## 如何选型：优势、局限与工程权衡

位置编码之所以有效，是因为它为缺乏顺序偏置的注意力结构补充了序列坐标系。

### 主流 Positional Encoding 机制总览

若把全文涉及的几类主流方案先放到同一张表里，可以更容易建立整体坐标系：

| 机制 | 位置信息写入位置 | 更强调什么 | 长度外推 | 实现复杂度 |
| --- | --- | --- | --- | --- |
| 绝对位置编码 | 输入表示 | 绝对坐标 | 中等，视方案而定 | 低 |
| 相对位置编码 | 注意力打分或偏置 | 相对距离 | 较好 | 中 |
| RoPE | query / key 旋转 | 绝对位置 + 相对位移 | 较好 | 中 |
| ALiBi | 注意力分数上的线性偏置 | 距离先验与简洁外推 | 较好 | 低 |

### 按任务目标选型

若把位置机制的选择看成一个“任务约束驱动”的问题，那么一个更实用的判断方式是先回答：模型最需要的到底是绝对坐标、相对距离，还是长上下文下的稳定外推。

| 任务目标 | 更适合优先考虑的机制 | 主要原因 | 典型注意点 |
| --- | --- | --- | --- |
| 强调固定位置语义 | 绝对位置编码 | 句首、句尾、固定槽位等绝对坐标更直接 | 超出训练长度时外推通常较弱 |
| 强调局部关系与距离模式 | 相对位置编码 | 更容易复用“左侧一格”“右侧两格”这类结构 | 实现与注意力打分耦合更深 |
| 强调生成式建模与主流 LLM 兼容性 | RoPE | 与 query / key 内积结构结合紧，工程生态成熟 | 超长上下文常需插值、缩放或校准 |
| 强调超长外推的规则简洁性 | ALiBi | 线性距离偏置更容易直接延伸到更长序列 | 几何表达较弱，远距惩罚设计需谨慎 |

若进一步压缩成几条经验判断，可以得到：

- 当任务明显依赖“绝对位置编号”时，绝对位置编码往往更直接；
- 当任务更依赖可平移复用的距离模式时，相对位置机制通常更自然；
- 当任务是现代自回归语言模型，并且希望兼顾表达能力、缓存友好性与工程成熟度时，RoPE 往往是默认起点；
- 当任务首先追求更平滑的长度外推，而愿意接受较简单的位置归纳偏置时，ALiBi 是值得考虑的替代方案。

### 常见变体

| 变体 | 核心思想 | 典型优点 | 典型代价 |
| --- | --- | --- | --- |
| 可学习绝对位置编码 | 为每个位置单独学习 embedding | 灵活 | 长度外推弱 |
| 正弦余弦位置编码 | 用解析函数生成多频率位置向量 | 无需学习、具一定外推性 | 绝对位置主导 |
| 相对位置编码 | 把距离写入打分或偏置 | 强化距离建模 | 实现复杂度上升 |
| RoPE | 对 query / key 做旋转 | 与内积耦合紧、适合 LLM | 频率设计影响外推 |
| ALiBi | 用线性距离偏置替代显式旋转或查表 | 简洁、长序列友好 | 表达能力与适配方式较依赖任务 |

### 主要优势

- **补足顺序信息**：使模型区分同一组 token 的不同排列。
- **保留全局建模优势**：无需像 RNN 那样串行处理，也能感知位置。
- **设计空间丰富**：可根据任务选择绝对、相对、旋转等不同形式。
- **更适配长距离依赖**：尤其相对位置与 RoPE 更适合长上下文场景。

### 典型局限

- **绝对位置编码外推有限**：训练长度之外的位置可能表现不稳。
- **任务所需的顺序偏置并不相同**：文本、语音、图像 patch 的位置结构差异很大。
- **相对位置实现更复杂**：会改变 attention 的打分形式与工程实现。
- **长上下文扩展并不只靠位置编码**：还受注意力复杂度、归一化、缓存与训练分布影响。

### 常见失效模式

- 长度远超训练分布时，模型对后段 token 的位置感知变弱；
- 位置信号过强时，模型可能过度依赖表面顺序模式；
- 位置信号过弱时，模型又会退化为「只看词、不看顺序」的内容匹配器。

### 工程上的常见干预手段

- 为长上下文任务使用相对位置编码或 RoPE；
- 采用位置插值、频率缩放等方式扩展上下文窗口；
- 在实现中配合 KV cache，避免推理时重复计算全部位置信号；
- 根据任务是否强调局部距离、全局坐标或长度外推来选择不同位置机制。

---

## 小结

Positional Encoding 的最关键思想，是为原本只擅长内容匹配的 attention 机制加入位置信号，使模型能够同时理解「是什么」与「在哪里」。

从形式上看，它既可以作为输入层的绝对坐标，也可以作为 attention 中的距离偏置，还可以像 RoPE 一样直接写入 query / key 的几何结构。现代模型中，位置编码通常承担顺序注入、距离建模与长上下文扩展这三类核心作用。
