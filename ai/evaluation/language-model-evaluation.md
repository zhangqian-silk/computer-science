## 符号约定与核心公式

本文统一使用以下记号：

| 符号 | 含义 |
| --- | --- |
| \(w_t\) | 序列中位置 \(t\) 的词 |
| \(V\) | 词表 |
| \(\lvert V\rvert\) | 词表大小 |
| \(P(w_t\mid w_{<t})\) | 语言模型给出的条件概率 |
| \(\mathcal{D}_{test}\) | 测试集 |
| \(N\) | 测试集中的 token 总数 |
| \(\mathcal{L}\) | 平均负对数似然 |
| \(\mathrm{PPL}\) | 困惑度（Perplexity） |
| coverage | 测试样本被训练集直接覆盖的程度 |

核心公式如下：

1. 测试集平均负对数似然：
\[
\mathcal{L}=-\frac{1}{N}\sum_{t=1}^{N}\log P(w_t\mid w_{<t})
\]

2. 困惑度：
\[
\mathrm{PPL}=\exp(\mathcal{L})
\]

3. 离散 \(n\)-gram 组合规模：
\[
\lvert V\rvert^n
\]

## 评估对象与核心问题

语言模型评估的核心，不只是看模型是否给训练集高概率，而是看它在测试集上是否仍能对真实文本分配合理概率。若一个模型只会记住训练语料中出现过的字符串，那么它在面对未见过的上下文时就会迅速失效。

因此，语言模型评估通常围绕三个问题展开：

- **预测是否准确**：模型给真实下一词的概率是否足够高；
- **未见组合是否可处理**：模型在训练中没见过的上下文上是否仍能给出合理估计；
- **改进来自哪里**：性能提升究竟是因为记忆更多频率，还是因为学到了可迁移的结构。

这三个问题分别对应常见的困惑度、覆盖率与泛化实验。

---

## 困惑度：概率分配质量的总体指标

困惑度（Perplexity, PPL）是语言模型最常见的总体评估指标。它本质上是测试集平均负对数似然的指数形式：

\[
\mathrm{PPL}=\exp(\mathcal{L})
\]

若模型总是给真实下一词更高概率，则平均负对数似然更小，PPL 也更低。因而，PPL 越低，通常表示模型对测试文本的概率分配越准确。

从直观上看，困惑度可以理解为模型在预测下一个词时的平均“不确定度”。若一个模型在每一步都只能把概率平均分给很多候选词，它就会表现得更“困惑”；若它能稳定把较高概率集中到真实词上，困惑度就会降低。

但 PPL 也有边界：

- 它依赖 tokenization、词表和测试集定义，跨实验时不能脱离设置直接比较；
- 它衡量的是概率分配质量，不直接等价于下游任务表现；
- 它能告诉我们模型“整体更准了”，但不直接说明改进究竟发生在哪类样本上。

因此，困惑度通常与覆盖率、错误分析和泛化实验配合使用。

## 覆盖率与零频率问题

在离散语言模型中，数据稀疏首先体现为覆盖率不足：测试集里有相当一部分 \(n\)-gram，在训练集中从未直接出现。若词表大小为 \(\lvert V\rvert\)，则一个 \(n\)-gram 模型面临的潜在组合规模约为

\[
\lvert V\rvert^n
\]

例如，当 \(\lvert V\rvert=10{,}000\) 且 \(n=4\) 时，潜在组合规模已达到 \(10^{16}\) 量级。真实语料只能覆盖其中极小部分，因此，大量测试上下文在训练阶段没有可靠计数。

这带来两个直接后果：

- **零频率陷阱**：未见过的高阶组合会得到 0 概率，或只能依赖回退与平滑；
- **高阶覆盖恶化**：随着 \(n\) 增大，测试集未覆盖比例会快速上升。

可用示意表概括这一趋势：

| 模型阶数 | 测试集中未见过的 \(n\)-gram 比例（示意） |
| --- | --- |
| **1-gram** | 较低，主要来自罕见词或未登录词 |
| **2-gram** | 明显上升 |
| **3-gram 及以上** | 常有相当一部分测试上下文未被训练集直接覆盖 |

覆盖率分析的重要性在于，它能帮助区分两类模型行为：

- 一类模型主要依赖“是否见过完全相同的字符串”；
- 另一类模型则能借助参数共享或连续表示，把未见样本映射到已见样本附近处理。

## 泛化实验：测试可迁移规律

泛化实验关注的不是训练集记忆，而是模型在未见样本上的行为。常见做法包括：

- **未见组合测试**：故意观察训练中未出现、但在语义或句法上合理的上下文；
- **替换实验**：把句子中的词替换为近义词、同类词或功能相近词；
- **长尾样本分析**：单独统计罕见搭配、低频词或低覆盖上下文上的表现。

例如，若训练中见过「猫 在 沙发 上 睡觉」，但没见过「豹子 在 沙发 上 睡觉」，则一个纯离散模型往往只能依赖回退和平滑；而一个具有连续表示的模型则可能利用「豹子」与「猫」的分布相似性，给前者分配更合理的非零概率。

这类实验的意义在于，它们能帮助判断模型是否超越了纯字符串记忆。若模型在相似替换后仍能保持相对稳定的概率估计，就说明它至少学到了一部分可迁移的局部规律。

---

## 连续覆盖与离散覆盖

覆盖率并不只是一张“见过 / 没见过”的二值表。对于连续表示模型，更关键的是样本在特征空间中的邻近关系。

- **离散覆盖**：只要字符串没有完全出现过，就视为未覆盖；
- **连续覆盖**：即便某个具体字符串未直接出现，只要它与已见样本在表示空间中足够接近，模型仍可能给出合理估计。

这一区别解释了为何连续表示模型常在高稀疏区域优于传统 \(n\)-gram：它们并没有消除未见样本，而是改变了“如何处理未见样本”的方式。

---

## 使用边界

这些评估方法是语言模型的通用分析工具，而不是某个具体模型专属的章节。它们可用于 `N-gram`、`NPLM`、`RNN`、`LSTM`、`Transformer` 等不同模型。

在具体模型文档中，更合适的做法通常是：

- 简述该模型为什么会在这些指标上受益或受限；
- 需要系统解释时，再引用单独的评估文档。
